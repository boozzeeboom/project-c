## Баг 1: Potential Shift+W конфликт с обычным thrust

**Серьёзность:** Средняя | **Приоритет:** P1 | **Статус:** Ожидает тестирования

### Описание
Shift+W активирует MODULE_MEZIY_THRUST (ускорение), но W также используется для обычного thrust. Есть риск что:
- Обычный thrust от W продолжает работать
- Meziy thrust boost добавляется поверх
- Результат: двойной thrust когда зажат Shift+W

### Текущее поведение (ожидаемое)
```
W только:          avgThrust * thrustForce * moduleThrustMult
Shift+W:           avgThrust * thrustForce * moduleThrustMult + meziyThrustBoost
```
Это **правильное** поведение — meziy boost должен быть ДОПОЛНИТЕЛЬНЫМ ускорением.

### Как проверить
```
1. Play Mode → зажать W → замерить скорость
2. Зажать Shift+W → скорость должна быть заметно выше
3. Если скорость такая же → баг (meziy thrust не работает)
4. Если корабль失控 → баг (слишком сильный thrust)
```

## Баг 2: MODULE_MEZIY_THRUST расход топлива

**Серьёзность:** Средняя | **Приоритет:** P2 | **Статус:** Ожидает балансировки

### Описание
`meziyFuelCost = 4` при continuous mode → `4 * 2 = 8 fuel/sec`.
При `maxFuel ~30` (Medium класс) → 3.7 секунды непрерывной работы.
Это может быть слишком мало для gameplay.

### Варианты решения
A. Увеличить `maxFuel` корабля
B. Уменьшить `meziyFuelCost` до 2-3
C. Принять как есть (короткий burst — это дизайн)

### Рекомендация
Протестировать в Unity. Если 3.7 сек мало → уменьшить `meziyFuelCost` до 3.

## Баг 3: MeziyStatusHUD может не найти FuelSystem

**Серьёзность:** Низкая | **Приоритет:** P3 | **Статус:** Обработан

### Описание
`MeziyStatusHUD` ищет `ShipFuelSystem` через `FindObjectsByType`. Если на корабле нет FuelSystem → HUD покажет "Fuel: N/A".

### Обработка
Код уже проверяет `if (fuelSystem != null)` перед отображением топлива.
