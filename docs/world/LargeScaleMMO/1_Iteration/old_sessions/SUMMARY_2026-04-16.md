## 1. Зачем нужен большой мир (100 000+)

### Проблема
Unity по умолчанию работает с координатами в float32:
- **Точность:** ±10⁷ (20 000 000) единиц
- **Проблема:** При больших расстояниях от origin (0,0,0) возникают:
  - Дрожание объектов (jitter)
  - Z-fighting (мерцание близких поверхностей)
  - Плавающие точки (floating point errors)
  - Физические артефакты

### Решение: Floating Origin
Мир разделён на "чанки" вокруг origin. Игрок всегда near (0,0,0), а мир "плывёт" вокруг него.

```
                ┌─────────────────────────┐
                │     World Space          │
                │                         │
                │   ┌─────┐               │
                │   │Chunk│ ← Far objects  │
                │   │  B  │   offset -50km │
                │   └─────┘               │
                │                         │
                │        ┌─────────┐       │
                │        │  Origin │ ← Player
                │        │  (0,0,0)│   always here
                │        └─────────┘       │
                │                         │
                │   ┌─────┐               │
                │   │Chunk│ ← Near objects │
                │   │  A  │   offset 0     │
                │   └─────┘               │
                └─────────────────────────┘
```

## 3. Как это работает

### 3.1 Floating Origin Algorithm

```csharp
// Pseudocode
void Update()
{
    // Distance from origin
    float distance = player.position.magnitude;
    
    // Threshold: when to shift (e.g., 50,000 units)
    const float SHIFT_THRESHOLD = 50000f;
    
    if (distance > SHIFT_THRESHOLD)
    {
        // Calculate offset
        Vector3 offset = CalculateOptimalOffset(player.position);
        
        // Apply to all world objects
        ShiftWorldObjects(offset);
        
        // Apply to all players
        ShiftAllPlayerPositions(offset);
        
        // Log for debugging
        Debug.Log($"[FO] Shifted world by {offset}");
    }
}
```

### 3.2 Chunk Loading Strategy

```
Chunk Size: 10,000 x 10,000 x 10,000 units

┌─────┬─────┬─────┐
│  -1 │  0  │  1  │  ← Chunk X
├─────┼─────┼─────┤
│ -1,-1│ 0,-1│ 1,-1│
├─────┼─────┼─────┤
│ -1,0 │ 0,0 │ 1,0 │  ← Origin Chunk (player always here)
├─────┼─────┼─────┤
│ -1,1 │ 0,1 │ 1,1 │
└─────┴─────┴─────┘

Load Distance: 2 chunks in each direction
Active Chunks: 5x5 = 25 chunks max in memory
```

### 3.3 Streaming Events

| Event | Описание |
|-------|----------|
| `OnChunkApproaching` | Chunk soon will be loaded |
| `OnChunkLoaded` | Chunk data loaded |
| `OnChunkUnloading` | Chunk data can be freed |
| `OnOriginShift` | Origin shifted (for UI feedback) |
