## Implementation Completed

### 1. ServerStormManager.cs ✅

**Location:** `Assets/_Project/Scripts/Core/ServerStormManager.cs`

**Features:**
- Server-authoritative (IsServer check)
- Manages 5 storms at world-space positions
- Storm positions synchronized via ClientRpc
- Storms move with wind (reads from WindManager.Instance on server)
- Lightning triggers randomly (10-30 second intervals)
- Lightning events sent via TriggerLightningClientRpc

**Key Methods:**
- `SpawnInitialStorms()` — spawns 5 storms at random world positions
- `Update()` — moves storms with wind, handles lightning timing
- `StormSpawnClientRpc()` — notifies clients of new storms
- `SyncStormStatesClientRpc()` — periodic position sync at 0.5 Hz
- `TriggerLightningClientRpc()` — triggers lightning effect on client

### 2. StormController.cs ✅

**Location:** `Assets/_Project/Scripts/Core/StormController.cs`

**Features:**
- Client-side visual controller
- Receives state from ServerStormManager
- Lightning VFX + material flash effect
- Visibility culling at 50km

- `UpdateState(Vector3 worldPos, float intensity)` — receive server update

## Architecture Integration

### Scene Hierarchy (BootstrapScene)

```
BootstrapScene.unity (Never unloaded)
├── WindManager                              ← Central wind
├── ServerWeatherController                  ← Broadcasts wind to clients
├── ServerStormManager (NEW)                ← 5 server storms
├── StormController (5 instances)            ← Client visuals (DontDestroyOnLoad)
└── CloudManager                             ← Near clouds, distant, veil
```

### Wind Flow (Fixed)

```
ServerWeatherController.Update()
    ↓
BroadcastWindClientRpc() → clients
    + ApplyWindToLocal() → WindManager.Instance (server)
    ↓
ServerStormManager.Update() reads WindManager.Instance.CurrentWindDirection/Speed
    ↓
Storms move with wind
    ↓
SyncStormStatesClientRpc() → clients
    ↓
StormController.UpdateState()
```

### Storm Sync Flow

```
Server: SpawnStorm()
    → StormSpawnClientRpc(id, position, intensity)
    → Clients: StormController.Initialize()

Server: Update() moves storms
    → SyncStormStatesClientRpc() every 2s
    → Clients: StormController.UpdateState()

Server: Lightning timer fires
    → TriggerLightningClientRpc(stormId)
    → Clients: StormController.TriggerLightning()
```

## Wind Flow Investigation (2026-05-07)

### Issue: Wind speed drops to 0 and never recovers

**Root Cause:** `_windSpeed = 0f` default in code. If scene value not loaded properly OR if variation formula produces 0, speed stays at 0 forever because `newSpeed = _windSpeed * speedMod = 0 * anything = 0`.

**Fix Applied:**
1. `ServerWeatherController.ApplyWindVariation()`: Changed minimum clamp from `0f` to `1f`
```csharp
_windSpeed = Mathf.Clamp(newSpeed, 1f, 100f);  // Was 0f
```

2. `WindManager.Update()`: Changed minimum threshold from `0.01f` to `1f`
```csharp
if (CurrentWindSpeed < 1f) CurrentWindSpeed = 1f;  // Was 0.01f -> 0f
```

---

### Issue: NullReferenceException in ServerStormManager

**Root Cause:** `NetworkManager.Singleton` can be null even when `IsServer` is true.

**Fix Applied:**
```csharp
if (NetworkManager.Singleton == null || !NetworkManager.Singleton.IsListening)
{
    return;
}
```

---

### Issue: Storms not visible - ClientControllers registration bug

**Root Cause:** OnEnable() pre-registered with StormId=0 (default), Initialize() couldn't match ID from server.

**Fix Applied:** OnEnable() now empty, Initialize() is only registration point.
