Two-Layer Architecture

SCENE LAYER (79,999×79,999)
  ↓ uses
CHUNK LAYER (2,000×2,000)
  • Scene = Unity Scene file
  • Chunk = procedural content
  • 625 chunks per scene (40×40)
  • 9 scenes loaded max (3×3)

Network Architecture

NetworkPlayer Prefab

  • NetworkObject - NGO identity
  • NetworkTransform - client authority
  • NetworkPlayer - movement/trade
  • CharacterController
  • PlayerInputReader

Note: Never shifted by FloatingOriginMP

Scene Management

  • ServerSceneManager - tracks client→scene
  • ClientSceneLoader - loads 3×3 neighborhood
  • SceneBoundNetworkObject - visibility
  • SceneTransitionCoordinator - RPC

Scene Transition Flow

1. Player crosses boundary (e.g., x > 79,999)
→ PlayerSceneTracker detects scene change
→ ServerSceneManager.TransitionClient()
→ NetworkHide(clientId) on old scene objects
→ LoadSceneClientRpc → ClientSceneLoader
→ Client loads new scene + neighbors (3×3)
→ NetworkShow(clientId) on new scene objects
→ Player spawns at local position ~0

NGO CheckObjectVisibility

Key insight: Called at spawn time only.

  • Returns false → object never spawns on that client
  • No traffic, no memory garbage
  • For runtime changes: NetworkHide() / NetworkShow()
