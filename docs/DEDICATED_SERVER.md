# Dedicated Server — Руководство по запуску

## Что это

**Dedicated Server** — режим запуска сервера без клиентской части (без рендеринга, без ввода). Сервер только обрабатывает сетевые подключения и синхронизацию.

## Как запустить

### Вариант 1: Через Unity Editor

В главном меню появится кнопка **"Start Server"** (красная, рядом с Host и Client).

### Вариант 2: Командная строка (Headless режим)

```bash
# Windows
ProjectC.exe -batchmode -nographics -server

# Linux
./ProjectC.x86_64 -batchmode -nographics -server
```

**Параметры:**
- `-batchmode` — запуск без окна Unity
- `-nographics` — без инициализации графического устройства (экономит ресурсы)
- `-server` — флаг Dedicated Server (автоматически запускает сервер)

### Вариант 3: Build сценарий (автоматизация)

Создайте `.bat` файл для Windows:

```bat
@echo off
echo Запуск Dedicated Server Project C...
start "" "Builds\ProjectC.exe" -batchmode -nographics -server
echo Сервер запущен на порту 7777
pause
```

## Настройки

### Порт сервера

По умолчанию: `7777`

Изменить в Unity: `NetworkManagerController` → `Server Port`

### IP адрес

Для локального сервера: `127.0.0.1`  
Для внешнего: публичный IP или домен

## Подключение клиентов

Клиенты подключаются через кнопку **"Start Client"**, указав IP и порт сервера.

## Мониторинг

- Консоль сервера показывает логи подключений/отключений
- Статус обновляется в UI (если не headless режим)

## Известные ограничения

- [ ] Нет консоли администратора (в планах)
- [ ] Нет сохранения состояния при перезапуске
- [ ] Инвентарь не синхронизируется (Этап 2, в работе)

✅ **Готово** — кнопка в UI + launch args + автозапуск
