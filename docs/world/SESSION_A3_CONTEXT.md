## 🧪 План отладки (пошагово)

### Шаг 1: Debug-лог позиций

Добавить в `MountainMeshBuilder.BuildPeakMesh()`:
```csharp
Debug.Log($"[MountainMeshBuilder] {peakData.displayName}: " +
          $"worldPosition={peakData.worldPosition}, " +
          $"meshHeight={meshHeight}, " +
          $"gameObjectPosition={transform.position}, " +
          $"baseRadius={peakData.baseRadius}");
```

**Ожидаемый вывод для Эвереста:**
```
Эверест: worldPosition=(0, 8848, 0), meshHeight=442.4, gameObjectPosition=(0, 8626.8, 0), baseRadius=600
```

**Если вывод другой** — проблема в данных PeakData.
**Если вывод правильный, но визуально кучно** — проблема в рендеринге/камере.

### Шаг 2: Play mode проверка

1. Запустить Play mode
2. Переместить камеру рядом с горой (X=0, Y=8700, Z=0)
3. Проверить видна ли гора
4. Если НЕ видна — масштаб меша слишком маленький для расстояния камеры

### Шаг 3: Эксперимент с параметрами

В Inspector у MountainMeshBuilder:
- **heightScale:** попробовать 0.01 → 0.1 → 1.0
- **meshHeightMultiplier:** попробовать 5 → 50 → 100
- **baseRadius:** попробовать 600 → 200 → 100

**Наблюдать:** как меняется визуальный размер.

### Шаг 4: Проверить X,Z координаты

В Hierarchy выбрать разные горы, проверить Transform.position:
- Эверест: (0, ~8627, 0)
- Монблан: (-1310, ~4688, 2810)
- Кибо: (-1881, ~5748, -3010)

**Если все Y примерно одинаковые** — meshHeight вычисляется неправильно.
**Если все X,Z близки к 0** — координаты НЕ применяются.

### Шаг 5: Проверить Bounds меша

```csharp
Debug.Log($"Mesh bounds: {mesh.bounds.center}, size: {mesh.bounds.size}");
```

**Ожидаемо для Эвереста:** center=(0, 221, 0), size=(1200, 442, 1200)
