## CLOUDENGINE Analysis

### Why CLOUDENGINE Looks Volumetric

From deep analysis of `cloud_advanced.frag`:

1. **Volumetric accumulation** — color builds up over 100+ ray steps
2. **Density gradient normals** — lighting reacts to 3D density changes (lines 166-170)
3. **Rim lighting from view direction** — creates edge glow (lines 172-186)
4. **Ghibli cel-shading** — quantized lighting levels

### Key Insight
The "vortices and voids" are NOT from complex noise - they're from accumulation + normals + rim lighting. Simple 2D FBM + proper lighting = volumetric look.

## Architecture

### Current Implementation (Plane-Based)

```
CloudManager
├── HorizonVeilRenderer (old, deprecated but works)
├── VeilRaymarchMeshController (NEW)
│   └── Plane mesh at Y=800-1200, follows player XZ
│       └── VeilRaymarchMesh.shader (volumetric raymarch)
└── ... (other cloud layers)
```

### Lighting Flow (CLOUDENGINE-Style)

```
raymarch step at samplePos
    ↓
CalcNormal(samplePos) — density gradient
    ↓
CelShade(NdotL, 3.0) — quantized diffuse
    ↓
Rim = pow(1 - dot(viewDir, normal), 3.0) — edge glow
    ↓
accumulatedColor += cloudColor * (1-alpha) * absorption
```

## ⚠️ KNOWN LIMITATION: No Diving Under Veil

### Problem
Current implementation uses horizontal plane at Y=800-1200. This works when player is ABOVE the veil (flying over). But when player goes BELOW the veil (Y < 800):

1. Player is INSIDE the volumetric layer
2. Plane mesh is ABOVE the player (at Y=1200)
3. No surface to display inner veil wall
4. Result: empty space, no veil visible

### User Requirement
"若玩家在云层下方，就看不到内部的云壁"

This is a **critical architectural limitation** that requires separate solution.

### Potential Solutions (for future work)

#### Option 1: Cylinder Approach (Recommended)
Create a cylinder around the player:
```
       /‾‾‾‾‾‾‾‾\
      / |      | \
     /  | 800  |  \   ← player inside at Y=500
    /   | -    |   \     sees INNER surface of cylinder
   /    |1200  |    \
  /     |      |     \
 /______|______|______\
```

- Cylinder follows player (XZ)
- Height: 800-1200
- Inside rendering (Cull Front) so player sees inner wall
- Works for all vertical angles

#### Option 2: Multiple Planes
Use 1 horizontal + 4 vertical planes to approximate cylinder. Simpler but harder to seamless blend.

#### Option 3: True Volumetric Rendering
Render volumetric fog around player position using render-to-texture. Most complex but correct.

### Decision Required
This limitation should be documented as Phase 4a-INCOMPLETE requiring Phase 4c (Diving Under Veil).
