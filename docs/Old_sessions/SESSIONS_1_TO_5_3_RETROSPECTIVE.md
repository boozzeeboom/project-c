## Извлечённые уроки (Lessons Learned)

### ❌ НЕ делать

1. **НЕ создавать asmdef без полного анализа зависимостей**
   - 57 ошибок, полный откат. Для indie-проекта asmdef не нужен.

2. **НЕ коммитить без проверки компиляции в Unity**
   - Каждый билд нужно проверять в Editor до отправки коммита.

3. **НЕ реализовывать сложные системы без тестирования в Unity**
   - Сессия 5 создала 4 файла и ни один не работал корректно при первом запуске.
   - Нужно: создать → проверить в Unity → исправить → коммит.

4. **НЕ менять RPC сигнатуры**
   - `SubmitShipInputRpc`, `AddPilotRpc`, `RemovePilotRpc` — стабильны, не трогать.

5. **НЕ забывать про URP особенности**
   - ParticleSystem требует URP-совместимый шейдер
   - `renderer.enabled = false` недостаточно — нужен `Stop()` + `Clear()`

6. **НЕ использовать фиксированные значения для физики**
   - Roll force = 15 не работала для массы 1000+. Использовать `mass * coefficient`.

7. **НЕ делать управление неудобным**
   - LeftShift + WASD для мезиевых модулей → переназначено на C/V, Z/X, Shift+A/D
   - WASD должны работать для обычного пилотирования.

### ✅ Делать

1. **Проверять компиляцию в Unity после каждого изменения**
2. **Добавлять Debug логи для диагностики**
3. **Использовать `Mathf.SmoothDamp` для плавных переходов**
4. **Разделять passive/active режимы для модулей**
5. **Документировать каждый баг и решение**
6. **Использовать `OnValidate()` для Editor валидации**
7. **Кэшировать component references в `Awake()`**
8. **Тестировать Input System и Input Manager совместимость**

```
WASD свободны для обычного пилотирования
```
