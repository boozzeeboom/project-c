### Проблемы:
1. ✅ Горы РАЗБРОСАНЫ правильно (XZ координаты работают)
2. ✅ Горы на ЗЕМЛЕ (основание Y=0 работает)

5. ❌ CapsuleCollider = "сферы", нет правильной коллизии
6. ❌ Нужно "за основу взять конусы"

### Что пользователь сделал в редакторе:
- Увеличил **Scale компонента Mountains** (родитель в Hierarchy) до **60**
- После этого горы стали "хоть как-то приближены к реально правильным размерам"

## Анализ: что даёт scale 60

```
Текущие размеры (без scale):
  Эверест: meshHeight = 450, baseRadius = 250

При scale родителя = 60:
  Визуально: meshHeight = 450 * 60 = 27,000
             baseRadius = 250 * 60 = 15,000
  h/r = 1.8 (соотношение НЕ меняется от scale!)
```

**Ключевой вывод:** Пользователь нашёл что **h/r = 1.8 ПРАВИЛЬНОЕ**, но **абсолютный размер НУЖНО УВЕЛИЧИТЬ**.

**НО:** scale 60 — это костыль на родителе. Нужно исправить формулы.

---

## Правильная формула (A4)

### Вариант: meshHeight от worldPosition.y с правильным множителем

Пользователь: "при scale 60 размеры правильные"
- При scale 60: meshHeight_visual = 450 * 60 = 27,000
- Значит ПРАВИЛЬНЫЙ meshHeight = 27,000 (без scale!)

**НО:** meshHeight = 27,000 — это ОГРОМНОЕ число для Unity mesh.

**Альтернатива:** Может пользователь имеет в виду что-то другое под "scale 60"?

### Давайте проверим:

Если меш Эвереста при scale 1:
- meshHeight = 450 units
- Это 450 метров в игре (1 unit = 1 метр)
- Реальный Эверест = 8,848 метров
- 450 ≠ 8,848 → meshHeight СЛИШКОМ МАЛЕНЬКИЙ

Правильный meshHeight для "реалистичного" Эвереста:
- 8,848 метров / 100 (масштаб высот) = 88.48 units
- **НО:** 88.48 units — это ЕЩЁ МЕНЬШЕ чем 450!

**Вывод:** "Реалистичный" масштаб НЕ работает. Нужен "игровой" масштаб.

### Игровой масштаб (Ghibli-стиль)

Из WorldLandscape_Design.md §9.1:
> "Преувеличенные пропорции: Горы выше, скалы круче, чем в реальности. Scale Y × 1.2+ для главных пиков."

**Цель:** Горы должны быть ВИЗУАЛЬНО ВПЕЧАТЛЯЮЩИМИ, НЕ реалистичными.

**Эмпирический множитель из данных пользователя:**
- Текущий meshHeight = 450
- При scale 60 "правильно"
- Значит целевой meshHeight ≈ 450 * K где K даёт "правильный" вид

**Проверка:** Может K = 10?
- meshHeight = 450 * 10 = 4,500 units
- Это 4.5 км — ВПЕЧАТЛЯЮЩЕ для игры
- baseRadius = 250 * 10 = 2,500 units
- h/r = 1.8 (соотношение сохраняется!)

**ИЛИ** множитель от worldPosition.y:
- meshHeight = worldPosition.y * 67.5
- Эверест: 88.48 * 67.5 = 5,972 ≈ 6,000 units
- Это 6 км — ВПЕЧАТЛЯЮЩЕ

## 🎯 РЕКОМЕНДУЕМЫЙ ПОДХОД A4

### 1. Формула высоты

```csharp
// Множитель подобран эмпирически: 
// при meshHeight ≈ 5000-9000 units горы выглядят "правильно"
float VISUAL_MULTIPLIER = 67.5f;

meshHeight = worldPosition.y * VISUAL_MULTIPLIER;
// Эверест: 88.48 * 67.5 = 5,972 units
// Монблан: 48.08 * 67.5 = 3,245 units
```

### 2. baseRadius

Увеличить пропорционально:

| Пик | baseRadius | meshHeight | h/r |
|-----|-----------|-----------|-----|
| Эверест | 350 | 5,972 | 17.1 |
| Монблан | 250 | 3,245 | 13.0 |

**ПРОБЛЕМА:** h/r = 17 — это ИГЛА!

**Решение:** baseRadius тоже умножить на VISUAL_MULTIPLIER:

| Пик | baseRadius | meshHeight | h/r |
|-----|-----------|-----------|-----|
| Эверест | 350 * 67.5 = 23,625 | 5,972 | 0.25 |
| Монблан | 250 * 67.5 = 16,875 | 3,245 | 0.19 |

**ПРОБЛЕМА:** h/r = 0.25 — это БЛИН!

### Вывод: НЕЛЬЗЯ просто умножить на один множитель

**Правильный подход:**
- meshHeight и baseRadius должны ИМЕТЬ РАЗНЫЕ множители
- Чтобы h/r был 1.5-2.0

```csharp
// Для Эвереста h/r = 1.8:
meshHeight = worldPosition.y * 67.5;    // = 5,972
baseRadius = meshHeight / 1.8;           // = 3,318

// h/r = 5,972 / 3,318 = 1.8 ✅
```

### 3. Форма меша

**ТОЛЬКО CONE для всех пиков.**

Простой конус:
```csharp
for (ring = 0 to rings):
    t = ring / rings
    radius = baseRadius * (1 - t)  // Линейное сужение 1.0 → 0.0
    y = t * meshHeight
```

**НО:** t=1.0 → radius=0 — это ОСТРАЯ вершина ("пипка").

**Исправление:** Вершина НЕ в точке, а небольшая площадка:
```csharp
float topRadius = baseRadius * 0.05f;  // 5% от базы
radius = Mathf.Lerp(baseRadius, topRadius, t);
```

### 4. Коллайдеры

**MeshCollider вместо CapsuleCollider:**
```csharp
var meshCollider = gameObject.AddComponent<MeshCollider>();
meshCollider.sharedMesh = mesh;
meshCollider.convex = true;
```
