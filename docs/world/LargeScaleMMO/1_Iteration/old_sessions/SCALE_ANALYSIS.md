## 2. Единая система координат

### 2.1. МИР (позиции GameObject в сцене)

| Ось | Масштаб | Пример |
|-----|---------|--------|
| **X** | Игровой (1:2000) | Эверест→Монблан = 3100 units |
| **Z** | Игровой (1:2000) | Эверест→Монблан = 3100 units |
| **Y** | Метры (1:1) | Эверест = 8848, Завеса = 1200 |

### 2.2. МЕШ ГОРЫ (локальные координаты)

| Параметр | Формула | Эверест пример |
|----------|---------|---------------|
| **Mesh Height** | `realHeightMeters * 0.01f * meshHeightMultiplier` | 8848 × 0.01 × 5 = **442.4 units** |
| **Mesh Base Y** | `0` (локальная система) | 0 |
| **Mesh Top Y** | Mesh Height | 442.4 |
| **Base Radius** | Из PeakData.baseRadius | 600 units |

**Соотношение высота/радиус:** 442.4 / 600 = **0.74** — нормальная пропорция горы.

### 2.3. Позиционирование GameObject

| Параметр | Формула | Эверест |
|----------|---------|---------|
| **GameObject.position.X** | Из PeakData.worldPosition.x | 0 |
| **GameObject.position.Y** | `worldPosition.y - (meshHeight / 2)` | 8848 - 221 = **8627** |
| **GameObject.position.Z** | Из PeakData.worldPosition.z | 0 |
| **Вершина меша (world)** | GameObject.Y + meshHeight | 8627 + 442 = **8848** ✅ |
| **Основание меша (world)** | GameObject.Y | **8627** |

**Ключевое правило:** GameObject.position.Y должен быть смещён так, чтобы ВЕРШИНА меша совпадала с `peakData.worldPosition.y` (вершина горы в мировых координатах).

### 2.4. Коллайдер

| Параметр | Формула | Эверест |
|----------|---------|---------|
| **direction** | 1 (Y-axis) | 1 |
| **height** | `meshHeight * 0.9` | 398.2 |
| **radius** | `baseRadius * 0.8` | 480 |
| **center** | `(0, meshHeight * 0.45, 0)` | (0, 199, 0) |

### 2.5. Снеговая линия

| Параметр | Значение | Единицы |
|----------|----------|---------|
| snowLineY в PeakData | 5500 | Метры (мировая высота) |
| Проверка в AssignMaterials | `peak.worldPosition.y > snowLineY` | ✅ Корректно |
| Эверест: 8848 > 5500 | Снег назначен | ✅ |

## 4. Формулы преобразования

```
// МИР → МЕШ (для генерации)
meshHeight = realHeightMeters * heightScale * meshHeightMultiplier
meshHeight = realHeightMeters * 0.01 * 5.0

// МЕШ → МИР (для позиционирования GameObject)
gameObjectPosition.Y = worldPosition.Y - (meshHeight / 2)

// Вершина меша в мировых координатах (проверка)
meshTopWorldY = gameObjectPosition.Y + meshHeight = worldPosition.Y ✅

// Основание меша в мировых координатах
meshBaseWorldY = gameObjectPosition.Y = worldPosition.Y - (meshHeight / 2)

// Коллайдер
collider.height = meshHeight * 0.9
collider.center = Vector3(0, meshHeight * 0.45, 0)
collider.radius = baseRadius * 0.8
```

## 5. Исправленные баги

### Bug #1: Гора "висит в небе"
**Причина:** `transform.position = peakData.worldPosition` ставит центр меша на вершину горы.
**Решение:** Смещаем Y на `meshHeight / 2` вниз.

### Bug #2: Мелкие горы
**Причина:** `meshHeightMultiplier = 5` с `heightScale = 0.01` даёт `8848 * 0.01 * 5 = 442`. При радиусе 600 — h/r = 0.74, что нормально. Но без смещения меш строится от Y=8848 вверх.
**Решение:** Исправлено в Bug #1.

### Bug #3: Материалы не применяются
**Причина:** `GetRockMaterialForPeak()` всегда возвращает `_himalayanRock`. `AssignMaterials()` создаёт массив материалов но без sharedMaterial.
**Решение:** Исправить GetRockMaterialForPeak() и AssignMaterials().

### Bug #4: Secondary пики без realHeightMeters
**Причина:** PeakDataFiller ставит `realHeightMeters = 0` для вторичных пиков.
**Решение:** Использовать `worldPosition.y` как fallback.
