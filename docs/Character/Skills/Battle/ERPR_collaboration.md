## 1. Краткая сводка ERPR

ERPR = "Easiest Role Play Rules". Делится на 2 механики: **социальную** и **боевую**.

### 1.1 Социальная (стр. 2, 4)

- **ОД (Очки Действия) = 100 за «день»** (условное число). Списываются Мастером за действия: осмотреться = 20, перейти ров в дождь = 10, в сухую = 5. Если 0 ОД — персонаж садится отдыхать, восстановление за 1 день.
- **ОВ (Очки Веса) = 120**. Перегруз → -1 клетка движения за каждые лишние 10 ОВ, +5 ОД за каждое действие.
- **УЕ (валюта)** — Мастер вводит.
- **Мастер — рассказчик + судья**. Игрок может предлагать «механическую лошадь» в стимпанк-мире → Мастер решает, существует ли это.

### 1.2 Боевая (стр. 3, 5-8)

- **Поле боя — сетка квадратов** (или гексов по решению группы). Движение только по прямым (без диагоналей) для квадратов.
- **20 ОЗ (Очки Здоровья)**. При >50% потере — штрафы. При 0 — судьбу решает ГМ.
- **Защита** — базовая 0, добавляется экипировкой.
- **3 Секунды** = боевые AP на ход. Секунды списываются за: перемещение (1 сек), удар/выстрел (1-3 сек), навык (1-3 сек).
- **5 клеток** = базовая скорость перемещения.
- **Сила / Ловкость / Интеллект** = базовые 10, модификатор к броскам (опционально).

### 1.3 Формула атаки (стр. 7) — ЯДРО боевой

```
final_damage = (STR_mod + 1dN + weapon_damage) × hit_location × crit
```

где:
- `STR_mod` — Сила (или базовые 10, если не введена).
- `1dN` — кубик, **класс кубика зависит от оружия** (d6/d8/d10/d12/d20).
- `weapon_damage` — базовый урон оружия (Мастер задаёт).
- `hit_location` — бросок 1d4: 1-2 = ×0.5 (руки/ноги), 3 = ×1 (тело), 4 = ×2 (голова).
- `crit` — бросок 1d100 + crit_modifier: если ≥100, ×2.

Опционально (по решению группы): можно убрать hit_location и crit для упрощения.

### 1.4 Навыки (стр. 8) — формула

```
skill = (radius + cost_seconds + duration + damage + special + uses_per_fight)
```

Пример: «Удар Стихии Воды — радиус 3 кл, использование в радиусе 5 кл, урон ×2, 3 сек, 1 раз за бой». **Магия приведена как пример**, но в настолке она **факультативна** — определяется сеттингом. Сама система ERPR **безмагична по ядру**.

### 1.5 Кубики (стр. 9)

- **d100** — основной (проверка параметров)
- **d20, d12, d10, d8, d6** — модификаторы урона (по типу оружия)
- **d4** — hit location

## 3. Что переносимо прямо сейчас (без изменения scope)

### 3.1 Damage Dice (1dN) — нативная замена на Unity

ERPR: оружие определяет класс кубика (d6 меч, d8 алебарда, d12 двуручник). Project C может перенять это **напрямую**:

```csharp
// В WeaponItemData (расширение к T-CB03):
[Header("Damage (ERPR-style)")]
[Tooltip("Класс кубика урона: d6 / d8 / d10 / d12 / d20. См. docs/Character/Skills/Battle/ERPR_collaboration.md §3.1")]
public DamageDice damageDice = DamageDice.d6;  // default для меча

[Tooltip("Базовый урон оружия (без кубика и модификаторов)")]
[Range(0, 50)] public int baseDamage = 3;

[Tooltip("Модификатор крит-броска: 1d100 + critMod >= 100 → crit ×2")]
[Range(-20, 20)] public int critModifier = 0;

public enum DamageDice : byte { d4=4, d6=6, d8=8, d10=10, d12=12, d20=20 }
```

**Эффект навыков:** навык с `damageMultiplier = 2.0` («Удар Стихии Воды») → в Combat-движке: `final = (STR + 1dN + base) × hitLocation × crit × skillMult`. **Точная копия ERPR-формулы**, только `1dN` = `Random.Range(1, N+1)`.

**Что это даёт:**
- **Разнообразие оружия** без переписывания формул. Меч d6, алебарда d8, двуручник d10, спец. оружие d12.
- **Skill variance** через `damageMultiplier` (навыки-удары типа HeavySwing ×1.2).
- **Предсказуемость баланса** (d20 с большим baseDamage — медленное, но потенциально сильное).
- **Лор-совместимость**: всё описано в `02_LORE.md` (холодное, стрелковое, мезиевое) — можно прямо маппить.

### 3.2 Hit Location (1d4) — нативная замена

```csharp
// В Combat-движок (будущее):
public enum HitLocation : byte {
    Limbs = 0,    // 1-2 на d4 → ×0.5
    Torso = 1,    // 3 на d4 → ×1.0
    Head = 2,     // 4 на d4 → ×2.0
}

public static float GetMultiplier(HitLocation loc) => loc switch {
    HitLocation.Limbs => 0.5f,
    HitLocation.Torso => 1.0f,
    HitLocation.Head => 2.0f,
    _ => 1.0f,
};
```

**Лор-обоснование:** в `02_LORE.md §4` мы уже предложили damage-types (Physical/Ballistic/Antigrav/Explosive/Mesium). HitLocation — **дополнительный** множитель. Очень удачно ложится на 5 дисциплин из `20_SKILL_TREES.md`:
- **Melee** (ближний бой) — голова чаще при skilled-ударе (`aimed_shot` technique повышает шанс `Head`).
- **Ranged** (стрелковое) — Torso/Head, но Limbs — частый miss-смежный.
- **Antigrav** (спец. удары) — игнорирует hit_location? Или ×3 Head (аномальная g-волна).

### 3.3 Crit 1d100 — нативная замена

Прямая переносимость. `Random.Range(1, 101) + critModifier >= 100`. **Без изменений в формулах**.

**Лор-обоснование:** крит в ERPR — «удачное попадание в уязвимое место» (двойной урон). В Project C:
- **Антиграв-клинок** → crit_modifier = +10 (аномальное поле «притягивает» удар к уязвимости).
- **Мезиевое оружие** → crit_modifier = +5 (нестабильная газовая среда).
- **Обычный меч** → crit_modifier = 0.

**Это даёт навыкам реальный gameplay-impact**: Antigrav-ветка (`20_SKILL_TREES.md §4`) получает **не только proficiency**, но и **crit-усиление** через `WeaponTechniqueUnlock("anti_gravity_aura")` → +crit_modifier.

### 3.4 Сила/Ловкость/Интеллект — УЖЕ 1:1

ERPR-статы (10 базово, модификатор к броскам) ↔ Project C-статы (`StatsConfig`, `StatsWorld`). **Полное соответствие**, ничего не надо менять. Подтверждает, что мы в правильном направлении.

| **Кубики d4/d6/d8/d12/d20/d100** | В MMO заменяются `UnityEngine.Random.Range(1, N+1)`. Семантика та же, реализация — Unity-API. **Это не "не переносимо" — это "переносимо как реализация"** (см. §3.1). |

## 6. ВЫВОД: вписать можно, но с ограничениями

### 6.1 Короткий вердикт

**Да, ядро ERPR-боевой системы вписывается в Combat-ветку Project C v2 character progression**, а именно:
- **Damage dice 1dN** — нативно переносимо в `WeaponItemData` (T-CB03).
- **Crit 1d100** — нативно переносимо в Combat-движок.
- **Hit location 1d4** — опционально переносимо в Combat-движок.
- **Сила/Ловкость/Интеллект (модификаторы)** — уже 1:1 совпадают.
- **Защита от экипировки** — переносимо (новое поле `armorDefense` в `ClothingItemData`).

**Что НЕ вписывается** (см. §4):
- ❌ Сетка (3D мир).
- ❌ Пошаговость (real-time MMO).
- ❌ ГМ-свобода (сервер-авторитативный).
- ❌ Магия (lore запрет).
- ❌ ОД = 100/день (MMO respawn-loop).
- ❌ 20 ОЗ + "судьба у ГМ" (persistent health).

### 6.2 Что это даёт для Combat-ветки из `20_SKILL_TREES.md`

Если принять ERPR-пакет (damage dice + crit + hit location), то в `20_SKILL_TREES.md` можно обогатить каждую ноду **не только proficiency/StatMod**, но и **damage-механикой**:

| Навык | Damage-механика (опционально) |
|---|---|
| `melee_basic_sword` | `damageDice = d6`, `baseDamage = 3`, `critModifier = 0` |
| `melee_great_sword` | `damageDice = d10` (медленнее, но сильнее), `baseDamage = 4`, `critModifier = +5` |
| `melee_precision_strike` | technique: +20% шанс `HitLocation.Head` |
| `ranged_basic_crossbow` | `damageDice = d8`, `baseDamage = 4`, дистанция 5 клеток (= N метров) |
| `ranged_mesium_mastery` | technique: `crit × 3` (мезий нестабилен) |
| `antigrav_blade` | `damageDice = d8`, `baseDamage = 3`, `critModifier = +10` |

| `defense_master_defender` | technique: `counter_damage = +d4` при успешном парировании |

**Это превращает 31 навык из "пассивных бонусов" в "активную боевую механику"** — играбельную, прозрачную, балансируемую.
