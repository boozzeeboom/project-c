# План разработки ММО "Project C: The Clouds" на Unity

## Этап 0: Подготовка окружения (Неделя 1)
**Цель:** Настроить рабочую среду и базовую архитектуру проекта.

### Задачи:
1. **Установка ПО:**
   - Unity Hub + Unity 6 (или 2022.3 LTS) с модулями:
     - Universal Render Pipeline (URP)
     - Netcode for GameObjects (NGO) / Mirror (выбрать окончательно на этапе 1)
     - Visual Studio Code с расширениями: C#, Unity Debugger, GitLens
   - Git LFS для работы с ассетами
   - Docker (для локального тестирования сервера)

2. **Настройка репозитория:**
   - Структура папок: `Assets/Scripts`, `Assets/Prefabs`, `Server/`, `Docs/`
   - `.gitignore` для Unity + игнорирование тяжелых ассетов
   - Ветка `main` (стабильная), `dev` (разработка), `feature/*`

3. **Базовая архитектура:**
   - Решение по сетевому стеку: **Mirror Networking** (лучше для ММО) или **NGO + Custom Server**
   - Прототип серверной части: .NET 8 Console App с WebSocket/TCP
   - Документация протокола клиент-сервер (формат сообщений: JSON/Protobuf)

---

## Этап 1: Прототип ядра геймплея (Недели 2-4)
**Цель:** Реализовать базовые механики в оффлайн-режиме.

### Задачи:
1. **Контроллер персонажа:**
   - Режим полета на корабле (физика Rigidbody + аэродинамика)
   - Режим пешего перемещения (анимации, коллизии)
   - Плавный переход между режимами (посадка/взлет)

2. **Система облаков и мира:**
   - Процедурная генерация облачных островов (шум Перлина)
   - Объемные облака через URP Volumes + шейдеры
   - Базовое освещение "Ghibli-style" (мягкие тени, теплая палитра)

3. **Интерактивность:**
   - ✅ Подбор предметов (E — ближайший в радиусе 3м)
   - ✅ Круговой инвентарь (Tab — 8 секторов, группировка по типам)
   - ✅ Hover-подсветка и подсписок при >1 предмете
   - ⏳ Открытие сундуков (контейнеры с несколькими предметами)

**Результат:** Оффлайн-билд с работающим перемещением, миром и инвентарём.

**Версия:** `v0.0.6-inventory` (4 апреля 2026)

---

## Этап 1.5: Доработка инвентаря (Недели 5-6)
**Цель:** Улучшить визуал и функциональность системы предметов.

### Задачи:
1. **Визуальное улучшение колеса:**
   - «Облачный» дизайн в стиле Ghibli-эстетики
   - 3D/2D иконки предметов в секторах
   - Слот 9 (центр) — для ключевого/тяжёлого предмета

2. **Визуализация на персонаже:**
   - Подобранные предметы отображаются на модели персонажа
   - Экипировка видна другим игрокам (сетевая синхронизация)

3. **Удобная система добавления:**
   - Редактор для расстановки предметов в мире
   - Шаблоны наборов предметов (лут-таблицы)

**Результат:** Полированная система инвентаря с визуальной обратной связью.

---

## Этап 2: Сетевой фундамент (Недели 7-10)
**Цель:** Реализовать клиент-серверное взаимодействие.

### Задачи:
1. **Серверная часть:**
   - Архитектура: Авторитарный сервер (клиент только отправляет ввод)
   - Реализация:
     - WebSocket сервер на .NET 8 (или Mirror Dedicated Server)
     - Система комнат/лобби для групп до 4 игроков
     - Синхронизация позиций, анимаций, состояний объектов

2. **Клиентская интеграция:**
   - Замена локального контроллера на сетевой (NetworkBehaviour)
   - Предсказание движения (Client-side Prediction)
   - Интерполяция других игроков (Lerp + буферизация)

3. **Тестирование сети:**
   - Локальный запуск: 1 сервер + 2-3 клиента
   - Проверка синхронизации физики кораблей
   - Обработка разрывов соединения (переподключение)

**Результат:** Работающий мультиплеер с 2-4 игроками в одной сцене.

---

## Этап 3: Ролевая система и прогрессия (Недели 9-12)
**Цель:** Добавить RPG-элементы и сохранение данных.

### Задачи:
1. **Характеристики и навыки:**
   - Система прокачки (опыт, уровни, очки навыков)
   - Деревья навыков для пилотирования и выживания
   - Балансировка чисел (таблицы в ScriptableObject)

2. **Серверная база данных:**
   - PostgreSQL для хранения аккаунтов, прогресса, инвентаря
   - Redis для кэширования сессионных данных
   - API для безопасного обмена данными (авторизация через JWT)

3. **Инвентарь и экономика:**
   - Серверная валидация транзакций (защита от читов)
   - Торговля между игроками (если планируется)
   - Крафт предметов из ресурсов

**Результат:** Сохранение прогресса между сессиями, работающая прокачка.

---

## Этап 4: Контент и полировка (Недели 13-18)
**Цель:** Наполнить мир контентом и улучшить визуал.

### Задачи:
1. **Миссии и квесты:**
   - Система заданий (серверная логика выполнения)
   - Диалоги с NPC (ветвящиеся сюжеты)
   - Ежедневные испытания для удержания игроков

2. **Визуальные улучшения:**
   - Финальные шейдеры для облаков и воды
   - Анимации персонажей и кораблей (Mixamo/Custom)
   - Пост-обработка: bloom, color grading, хроматическая аберрация

3. **Звук и музыка:**
   - Динамический саундтрек (меняется от ситуации)
   - 3D-звуки двигателей, ветра, шагов
   - Голосовой чат (интеграция Vivox/Discord SDK)

**Результат:** Вертикальный срез игры с основным циклом геймплея.

---

## Этап 5: Масштабирование и оптимизация (Недели 19-24)
**Цель:** Подготовить инфраструктуру для большого онлайна.

### Задачи:
1. **Серверная масштабируемость:**
   - Шардинг мира (разделение на зоны/сервера)
   - Оркестрация через Kubernetes (автомасштабирование под нагрузкой)
   - Мониторинг: Prometheus + Grafana (метрики нагрузки, лаги)

2. **Оптимизация клиента:**
   - LOD-системы для кораблей и островов
   - Occlusion Culling для сложных сцен
   - Пулинг объектов (пули, эффекты, NPC)

3. **Античит:**
   - Серверная валидация всех критичных действий
   - Обнаружение аномалий (скорость, урон, телепорты)
   - Репорты игроков + автоматические баны

**Результат:** Стабильная работа при 100+ одновременных игроках на шард.

## Критические риски и меры:
1. **Сложность сетевой синхронизации физики**  
   → Начать с упрощенной модели, тестировать с первых недель.

2. **Читеры в экономике**  
   → Вся критичная логика только на сервере, логирование действий.

3. **Высокие требования к инфраструктуре**  
   → Использовать облачные решения (AWS GameLift, Azure PlayFab) для старта.

4. **Перегрузка контентом**  
   → Фокус на минимально жизнеспособный продукт (MVP) для раннего доступа.

## Инструменты и зависимости:
- **Клиент:** Unity 6, URP, Mirror/NGO, DOTween (анимации), Cinemachine (камера)
- **Сервер:** .NET 8, WebSocket/Netty, PostgreSQL, Redis, Docker/K8s
- **DevOps:** GitHub Actions (CI/CD), Sentry (ошибки), Prometheus (мониторинг)
- **Ассеты:** Quixel Megascans (окружение), Mixamo (анимации), FMOD/Wwise (звук)
