## 🏗️ Архитектура

### Объекты на сцене

```
Player (Capsule)
├── CharacterController (физика пешехода)
├── PlayerController (WASD + Space + Shift)
├── PlayerStateMachine (переключение F)
├── ThirdPersonCamera (камера)
└── MeshRenderer (визуал капсулы)

Ship_01 (Cube/Model) — тег "Ship"
├── Rigidbody (физика полёта)
└── ShipController (управление кораблём)

Ship_02, Ship_03... — дополнительные корабли с такой же структурой
```

### Как работает переключение

**Пешеход → Корабль (F):**
1. Player находит ближайший корабль (< 5м) по тегу `Ship`
2. `CharacterController.enabled = false` (игрок не физический)
3. `MeshRenderer.enabled = false` (игрок невидим)
4. `PlayerController.enabled = false`
5. `ShipController.enabled = true` (управление кораблём)
6. Камера переключается на корабль, дистанция увеличивается

**Корабль → Пешеход (F):**
1. Проверка: корабль на земле ИЛИ скорость < 2 м/с
2. Игрок телепортируется на палубу (`ship.GetExitPosition()`)
3. `ShipController.enabled = false`
4. `CharacterController.enabled = true`
5. `MeshRenderer.enabled = true`
6. `PlayerController.enabled = true`
7. Камера возвращается к игроку

## ⚙️ Настройка нового корабля

### Шаг 1: Создать объект корабля
```
1. Правый клик в Hierarchy → 3D Object → Cube (или модель)
2. Назвать: Ship_01, Ship_Barca и т.д.
3. Поставить на платформу (Y = 1.5)
```

### Шаг 2: Добавить компоненты
```
1. Tag: Ship (создать через Add Tag если нет)
2. Add Component → Rigidbody
   - Use Gravity: ✅
   - Linear Drag: 1 (или настроить в ShipController)
   - Angular Drag: 2
3. Add Component → ShipController
   - По умолчанию выключен (enabled = false в Awake)
   - Настроить thrustForce, yawForce и т.д.
```

### Шаг 3: Настроить ShipController (в Inspector)
| Параметр | Значение | Описание |
|----------|----------|----------|
| Thrust Force | 500 | Сила тяги вперёд/назад |
| Max Speed | 30 | Максимальная скорость (м/с) |
| Yaw Force | 30 | Сила поворота (A/D) — медленная как у баржи |
| Pitch Force | 40 | Тангаж (мышь) |
| Vertical Force | 300 | Подъём/снижение (Q/E — лифт) |
| Anti Gravity | 1.0 | 0 = падает, 1 = зависает |
| Linear Drag | 1 | Сопротивление воздуха |
| Stabilization Force | 50 | Возврат к горизонту |

### Шаг 4: Готово
Корабль автоматически определяется PlayerStateMachine по тегу `Ship`.

## 📋 Управление

### Пешеход
| Клавиша | Действие |
|---------|----------|
| W/S | Вперёд/назад |
| A/D | Стрейф |
| Мышь | Вращение камеры |
| Space | Прыжок |
| Left Shift | Бег |
| **F** | Сесть в корабль |

### Корабль
| Клавиша | Действие |
|---------|----------|
| W/S | Тяга вперёд/назад |
| A/D | Рыскание (поворот) |
| Q/E | Вниз/Вверх (лифт) |
| Мышь Y | Тангаж (нос вверх/вниз) |
| Left Shift | Ускорение (x2 тяга) |
| **F** | Выйти из корабля |

## ⚠️ Известные проблемы

### 1. Управление кораблём примитивное
- Движения дёрганые, неестественные
- Нет плавного ускорения/торможения
- Корабль не «чувствуется» тяжёлым
- Стабилизация работает слишком резко

**Что нужно доработать:**
- [ ] Плавная интерполяция ввода (smoothing)
- [ ] Инерция массы — корабль не должен останавливаться мгновенно
- [ ] banking (автоматический крен при повороте)
- [ ] Ветровые течения как игровая механика
- [ ] Топливо (мезий) — ограниченное время полёта

### 2. Физика полёта
- Нет аэродинамического сопротивления на скорости
- Корабль может зависнуть на месте без энергии
- Нет ограничений по высоте (Завеса / мезий)
- Нет столкновений с горами/облаками

**Что нужно доработать:**
- [ ] Реалистичная аэродинамика (drag = f(velocity²))
- [ ] Лимит высоты (небесный барьер)
- [ ] Коллизии с миром (горы, облака)
- [ ] Урон при столкновении

### 3. Посадка/взлёт
- Нет анимации перехода
- Камера резко переключается
- Нет проверки ориентации корабля (можно сесть перевёрнутым)

**Что нужно доработать:**
- [ ] Плавная анимация появления/исчезновения капсулы
- [ ] Плавное переключение камеры
- [ ] Проверка: корабль горизонтален при посадке
- [ ] Звуковые эффекты двигателя

### 4. Множественные корабли
- Нет интерфейса выбора корабля (если рядом несколько)
- Нет разных классов кораблей (лёгкий/тяжёлый)
- Нет визуальных отличий

**Что нужно доработать:**
- [ ] UI список кораблей рядом
- [ ] Разные ShipController для разных классов
- [ ] Модели кораблей (не кубы)
