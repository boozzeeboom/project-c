#  Git Workflow для Project C

**Последнее обновление:** 2 апреля 2026 г.  
**Ветка:** `qwen-dev`

---

##  Основная концепция

Мы используем **систему веток** для безопасной разработки:

```
main (стабильная версия)
  └── dev (текущая разработка)
        └── feature/* (тестовые ветки для новых функций)
```

---

## 🎯 Структура веток

| Ветка | Назначение | Когда использовать |
|-------|------------|-------------------|
| **`main`** | Стабильная рабочая версия | Только релизы |
| **`dev`** | Текущая разработка | Основная рабочая ветка |
| **`qwen-dev`** | Эксперименты Qwen Code | Тестирование новых функций |
| **`feature/*`** | Конкретные функции | Например: `feature/cloud-system` |

---

## 🔄 Рабочий процесс

### 1. Перед началом работы

```bash
# Проверить текущую ветку
git branch

# Переключиться на рабочую ветку
git checkout qwen-dev

# Получить последние изменения
git pull upstream qwen-dev
```

---

### 2. Создание тестовой ветки (перед экспериментами)

```bash
# Создать новую ветку для тестов
git checkout -b test/cloud-system-1

# Или для новой функции
git checkout -b feature/cloud-system
```

**Именование веток:**
- `test/` — быстрые тесты
- `feature/` — новые функции
- `fix/` — исправления ошибок
- `docs/` — документация

---

### 3. Внесение изменений

1. **Делаешь изменения** в Unity/Qwen Code
2. **Проверяешь** что работает
3. **Коммитишь** изменения

```bash
# Посмотреть изменения
git status

# Добавить все файлы
git add .

# Или добавить конкретные файлы
git add Assets/_Project/Scripts/Core/CloudSystem.cs

# Сделать коммит
git commit -m "Добавлена система облаков (3 слоя)"
```

---

### 4. Если что-то пошло не так — откат!

#### Вариант А: Откатить последние изменения (до коммита)

```bash
# Отменить все изменения в файлах
git restore .

# Отменить изменения в конкретном файле
git restore Assets/_Project/Scripts/Core/CloudSystem.cs
```

#### Вариант Б: Откатиться к последнему коммиту

```bash
# Отменить все изменения и коммиты
git reset --hard HEAD
```

#### Вариант В: Откатиться к конкретной версии

```bash
# Посмотреть историю коммитов
git log --oneline -10

# Откатиться к коммиту (замени abc1234 на нужный хэш)
git reset --hard abc1234
```

---

### 5. Сохранение прогресса на GitHub

```bash
# Отправить ветку на GitHub
git push upstream qwen-dev

# Или если ветка новая
git push -u upstream feature/cloud-system
```

## 📦 Шпаргалка: основные команды

| Команда | Что делает |
|---------|------------|
| `git status` | Показать изменения |
| `git branch` | Список веток |
| `git checkout <branch>` | Переключить ветку |
| `git checkout -b <new>` | Создать и переключить |
| `git add .` | Добавить все файлы |
| `git commit -m "..."` | Сохранить изменения |
| `git push` | Отправить на GitHub |
| `git pull` | Получить с GitHub |
| `git log --oneline -10` | История коммитов |
| `git restore .` | Отменить изменения |
| `git reset --hard HEAD` | Откат к последнему коммиту |

## 🏷️ Система тегов (версионирование)

Для важных версий создаём **теги**:

```bash
# Создать тег
git tag v0.3.0-cloud-system

# Отправить теги на GitHub
git push upstream --tags
```

**Формат версий:** `v<major>.<minor>.<patch>-<description>`

Примеры:
- `v0.1.0-initial` — первая версия
- `v0.2.0-world-gen` — генерация мира
- `v0.3.0-cloud-system` — система облаков
- `v1.0.0-alpha` — альфа релиз

---

## 📋 Чек-лист перед коммитом

- [ ] Код компилируется без ошибок
- [ ] Игра запускается и работает
- [ ] Все изменения протестированы
- [ ] `.meta` файлы не сломаны
- [ ] Нет временных файлов (`*.disable`, `*.bak`)

---

## ⚠️ Что НЕ коммитить

| Файлы/Папки | Почему |
|-------------|--------|
| `Library/` | Кэш Unity, воссоздаётся |
| `Temp/` | Временные файлы |
| `Logs/` | Логи |
| `UserSettings/` | Локальные настройки |
| `*.disable`, `*.bak` | Временные файлы |
| `packages-lock.json` | Генерируется Unity |

## 🔧 Типичные сценарии

### Сценарий 1: Тестирование новой функции

```bash
# 1. Создать тестовую ветку
git checkout -b test/cloud-system

# 2. Сделать изменения в Unity/Qwen

# 3. Проверить что работает

# 4. Закоммитить
git add .
git commit -m "Тест: система облаков"

# 5. Если работает — влить в dev
git checkout qwen-dev
git merge test/cloud-system

# 6. Если не работает — удалить ветку
git checkout qwen-dev
git branch -D test/cloud-system
```

---

### Сценарий 2: Срочный откат

```bash
# Всё сломалось! Откатиться к последней рабочей версии:
git fetch upstream
git reset --hard upstream/qwen-dev
```

---

### Сценарий 3: Резервная копия перед экспериментами

```bash
# 1. Создать локальную резервную ветку
git checkout -b backup/2026-04-02

# 2. Вернуться на рабочую
git checkout qwen-dev

# 3. Если всё сломалось — вернуться к backup
git checkout backup/2026-04-02
```

## 💡 Советы

1. **Часто коммить** — каждые 15-30 минут или после каждой функции
2. **Понятные сообщения** — через неделю не вспомнишь что делал
3. **Тестовые ветки** — перед крупными изменениями
4. **Резервные копии** — перед рисковыми операциями
5. **Pull перед работой** — всегда начинай с `git pull`
