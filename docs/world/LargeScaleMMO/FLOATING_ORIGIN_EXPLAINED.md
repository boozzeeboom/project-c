# FloatingOriginMP — Как это работает

## Архитектура

```
Scene
├── TradeZones (статичный, всегда на 0,0,0)
├── WorldRoot (сдвигается)
│   ├── Mountains
│   ├── Clouds
│   ├── Farms
│   └── ...
├── FloatingOriginMP
└── NetworkPlayerSpawner → NetworkPlayer (НЕ внутри WorldRoot!)
```

## Принцип работы

**Цель:** Избежать потери точности float на больших координатах (>500k)

**Механизм:**
1. Игрок стоит на месте
2. Мир (WorldRoot) СДВИГАЕТСЯ вокруг игрока
3. TradeZones остаётся привязан к игроку

## Сдвиг мира

Когда игрок далеко от origin (> threshold):
- `WorldRoot.position -= offset` — сдвигает мир
- TradeZones ВОССТАНАВЛИВАЕТСЯ на (0,0,0)
- Игрок остаётся на месте (НЕ двигается с миром!)

## Результат

**До сдвига:**
```
Игрок: (1000000, 5, 0)
WorldRoot: (0, 0, 0)
TradeZones: (0, 0, 0)
```

**После сдвига (offset = 1M):**
```
Игрок: (1000000, 5, 0) ← остался на месте!
WorldRoot: (-1000000, 0, 0) ← сдвинулся
TradeZones: (0, 0, 0) ← восстановлен
```

**Визуально:** Игрок видит что горы/облака "уехали", но он остался рядом с TradeZones.

## Shift+R (ручной сброс)

1. Телепортирует игрока на (5, 5, 0) — рядом с TradeZones
2. `ResetOrigin()` — сдвигает WorldRoot на -1M
3. TradeZones остаётся на месте

**Результат:**
- Игрок рядом с TradeZones ✓
- Артефакты исчезают (игрок теперь на малых координатах) ✓
- Мир "уехал" — горы далеко ✗

## Вопрос: Это правильное поведение?

Если игрок должен оставаться рядом с горами — нужно переделать:
- Поместить игрока ВНУТРИ WorldRoot
- Тогда игрок будет двигаться вместе с миром

## Альтернатива: Server Authority

В multiplayer режиме обычно:
- Сервер управляет сдвигом мира
- Все клиенты получают синхронизированный сдвиг
- Игроки ВНУТРИ WorldRoot для синхронизации

## Текущая реализация

- `mode = OriginMode.Local` (однопользовательский)
- Игрок ВНЕ WorldRoot
- Shift+R — временное решение для тестирования
