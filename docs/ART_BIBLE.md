# Art Bible: Project C — The Clouds

**Версия:** 1.0 | **Дата:** 6 апреля 2026 г.
**Проект:** MMO на Unity 6, URP | **Стиль:** Sci-Fi + Ghibli
**По мотивам книги:** «Интеграл Пьявица» — Бруно Арендт

---

## 1. Визуальная идентичность

### 1.1. Ключевые принципы

1. **Sci-Fi + Ghibli** — промышленный дизайн (сталь, ржавчина, кирпич) + мягкие формы, градиенты, объёмный свет
2. **Нет магии** — всё технологично: антигравий, мезий, генераторы. Даже свечение — это физика, не волшебство
3. **Баржи, не истребители** — корабли тяжёлые, медленные, с инерцией. Это чувствуется в моделях и анимациях
4. **Мир над облаками** — небо, ветер, свобода. Но под ним — ядовитая Завеса с фиолетовыми молниями
5. **Утилитарная красота** — всё функционально, но красиво. Никакого декора ради декора

### 1.2. Anti-пи́ллары (чего этот проект НЕ делает)

- ❌ Не фотореализм — стилизация, мягкие формы
- ❌ Не тёмный мрачный постапокалипсис — тёплые закаты, свет, надежда
- ❌ Не киберпанк — неон, дождь, мегаполисы. Это **над** цивилизацией
- ❌ Не военные истребители — корабли-баржи, грузовые платформы
- ❌ Не средневековое фэнтези — технологии 2090 года, антигравий, электричество

### 1.3. MDA-эстетика (приоритет)

| Приоритет | Категория | Описание |
|-----------|-----------|----------|
| 1 | **Discovery** (Открытие) | Исследование мира над облаками, скрытые локации |
| 2 | **Sensation** (Ощущение) | Визуальная красота: закаты, облака, масштаб гор |
| 3 | **Fantasy** (Фантазия) | Быть пилотом в мире над облаками |
| 4 | **Fellowship** (Общение) | Кооп-полёты, торговля, гильдии |
| 5 | **Challenge** (Мастерство) | Навигация, управление кораблём-баржей |

### 1.4. Эмоциональная дуга сессии

```
Подключение → Величие (масштаб мира, вид на облака)
    → Исследование (любопытство, навигация между пиками)
        → Открытие (найти новую локацию, сундук, квест)
            → Уют (вернуться в город, торговая аллея, NPC)
```

| **Текст UI** | `#e0e0e0` | Светло-серый |

### 2.2. Цветовые правила

- **Тёплое сверху, холодное снизу** — небо тёплое (закат), Завеса холодная (фиолет)
- **Свечение = функция** — голубой = антигравий (безопасно), зелёный = мезий (опасно), фиолетовый = Завеса (смерть)
- **Закат — всегда** — даже «днём» небо имеет тёплый оттенок (атмосфера мезия рассеивает свет)

### 3.2. Post-Processing (URP Volume)

|--------|-----------|
| **Color Adjustments** | Post Exposure: +0.3, Contrast: +10, Saturation: +15 |
| **Bloom** | Threshold: 0.8, Intensity: 0.5, Soft: 0.7 |

- **God rays** — лучи света сквозь облака (фейковые: полупрозрачные плоскости или URP Volume)

- **Depth fog** — дальние пики в лёгкой дымке

## 4. Модели кораблей

### 4.1. Лёгкий корабль (Scout)

| Параметр | Значение |
|----------|----------|
| **Форма** | Торообразный (бублик), вытянутый по горизонтали |
| **Размеры** | ~6м длина × ~3м ширина × ~1.5м высота |
| **Вместимость** | 2-4 человека |
| **Скорость** | ~350 км/ч |
| **Полигонаж** | 5,000–8,000 tri (LOD0), 3,000 (LOD1), 1,000 (LOD2) |
| **Текстуры** | 1024×1024: Albedo + Normal + Emissive + AO |
| **Материалы** | 2–3: корпус (URP Lit), стекло кабины (прозрачный), emissive (двигатели) |
| **Детали** | 2-4 ветровые лопасти (вращающиеся), кокпит (прозрачный купол), антиграв-диски (свечение) |
| **Коллизия** | Упрощённый меш (box + capsule) |
| **Pivot** | В центре массы |

**Цветовая схема:** Белый/светло-серый корпус + медные/бронзовые акценты + голубое свечение двигателей

**Референсы:**
- «Небесный замок Лапута» (Ghibli) — мелкие корабли
- «Ветер крепчает» (Ghibli) — винты, лопасти
- Дирижабли 1930-х — обтекаемые формы

### 4.2. Средний корабль (Hauler)

| Параметр | Значение |
|----------|----------|
| **Форма** | Сигарообразный, закрытый |
| **Размеры** | ~13м длина × ~3м ширина |
| **Вместимость** | До 13 человек |
| **Скорость** | ~300 км/ч |
| **Полигонаж** | 12,000–18,000 tri (LOD0) |
| **Текстуры** | 2048×2048: Albedo + Normal + Emissive + AO |
| **Материалы** | 3–4: корпус, окна, emissive, детали |

### 4.3. Тяжёлый корабль (Freighter)

| Параметр | Значение |
|----------|----------|
| **Форма** | Открытая платформа, 1-3 яруса |
| **Размеры** | До 89м × 34м (Тяжёлый I) |
| **Скорость** | 150 км/ч |
| **Полигонаж** | 20,000–30,000 tri (LOD0) |
| **Текстуры** | 2048×2048 |

> **Для прототипа:** нужен только **Лёгкий корабль**. Остальные — позже.

### 4.4. Общие требования к кораблям

- **Плавные обтекаемые формы**, никаких острых углов
- **Видимые стыки панелей** корпуса (Ghibli-механизмы — «лапти», но красивые)
- **Антиграв-двигатели** — светящиеся кольца/диски (emissive, голубой glow)
- **Ветровые лопасти** — отдельные меши, вращение через код
- **UV-развёртка** — автоматическая + ручная подгонка швов
- **Формат** — FBX 2020+, Y-up
- **LOD-группы** — в одном FBX (маскированные)

- Cast/Receive Shadows: Off

## 11. Технические ограничения

### 11.1. URP-совместимость

- **НЕ использовать `Standard` шейдер** — только `Universal Render Pipeline/Lit`
- Кастомные шейдеры — на базе URP Unlit/Particles
- Post-processing — через URP Volume

### 11.2. Производительность (целевые значения)

| Метрика | Цель |
|---------|------|
| **Draw Calls** | < 2000 |
| **Tris per frame** | < 2M |
| **Shadow Distance** | 150м |
| **Shadow Cascades** | 2 |
| **Anti-Aliasing** | FXAA |
| **Renderer** | Forward (не deferred) |

### 11.3. Оптимизация

| Техника | Объекты |
|---------|---------|
| **GPU Instancing** | Облака, камни, здания |
| **Static Batching** | Статические объекты (пики, мосты) |
| **LOD Groups** | Корабли, пики, здания |
| **Occlusion Culling** | Сложные сцены |
| **Frustum Culling** | Автоматический |
