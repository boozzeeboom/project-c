## 🏗️ Архитектура

**Подход:** Один GameObject `Player` с переключением компонентов.

```
Player (GameObject)
├── PlayerController (пеший, CharacterController) — ВКЛ/ВЫКЛ
├── ShipController (корабль, Rigidbody) — ВКЛ/ВЫКЛ
├── ShipModel (3D модель корабля) — активен/скрыт
├── ThirdPersonCamera — всегда активна
└── PlayerStateMachine (управляет переключением)
```

**Почему один GameObject:**
- Камера всегда следует за одним объектом
- Не нужно спавнить/деспавнить префабы
- Проще сохранять состояние

## 📝 Порядок работы

### Шаг 1: ShipController — базовое управление кораблём ✅ ЗАВЕРШЁН

**Файлы:** `Assets/_Project/Scripts/Player/ShipController.cs`

**Что сделано:**
- MonoBehaviour (пока не сетевой)
- Rigidbody для физики
- Антигравитация — корабль зависает (компенсация гравитации)
- Управление:
  - W/S — тяга вперёд/назад (ветровые лопасти)
  - A/D — рыскание (медленное, как у баржи)
  - Мышь — тангаж (нос вверх/вниз)
  - Q/E — крен (минимальный, рамка стабилизирует)
- Камера автоматически переключается на shipDistance при активации
- Shift — ускорение x2

**Готовые решения Unity:**
- `Rigidbody.linearDamping` / `angularDamping` — аэродинамическое сопротивление
- `Rigidbody.useGravity` — гравитация включена
- `AddForce(transform.forward * thrust)` — тяга
- `AddForce(Vector3.up * antiGravity)` — компенсация гравитации

**Тест:** ✅ Корабль зависает, управляется, камера адаптируется

**Коммит:** ✅ `50cc193` — Шаг 1: ShipController — антигравитация

### Шаг 2: PlayerStateMachine — переключение режимов ✅ ЗАВЕРШЁН

**Файлы:** `Assets/_Project/Scripts/Player/PlayerStateMachine.cs`

**Что сделано:**
- Поиск кораблей по тегу `Ship` в радиусе 5м
- F — переключение пешком ↔ корабль
- При посадке: CharacterController + Renderer отключаются (игрок не физический)
- При выходе: игрок появляется на палубе (GetExitPosition)
- Проверка посадки: корабль на земле ИЛИ скорость < 2 м/с
- Камера автоматически переключается между игроком и кораблём
- Поддержка множества кораблей на сцене

**Готовые решения Unity:**
- `GameObject.FindGameObjectsWithTag()` — поиск кораблей
- `CharacterController.enabled` — вкл/выкл физики пешехода
- `Renderer.enabled` — вкл/выкл визуала
- `Behaviour.enabled` — вкл/выкл контроллеров

**Тест:** ✅ Подойти к кораблю → F → управление кораблём → F → выйти на палубе

**Известные проблемы:**
- ⚠️ Управление кораблём дёрганое (нет плавности)
- ⚠️ Нет banking (крен при повороте)
- ⚠️ Нет системы топлива
- ⚠️ Нет анимации перехода

**Коммит:** ✅ (будет после документации)

**Документация:** `docs/SHIP_SYSTEM_DOCUMENTATION.md`

### Шаг 3: Плавный переход (посадка/взлёт)

**Файлы:** `PlayerStateMachine.cs`, обновление `ThirdPersonCamera.cs`

**Что сделать:**
- Raycast вниз от игрока — проверка что можно сесть
- Условия посадки: высота < 2м, скорость < 1 м/с
- Плавное появление/исчезновения модели корабля (lerp scale)
- Камера адаптирует дистанцию: корабль = дальше, пеший = ближе

**Готовые решения Unity:**
- `Physics.Raycast()` с LayerMask
- `Mathf.Lerp()` / `Vector3.Lerp()` — плавные переходы
- Camera distance lerp — изменение дистанции камеры

**Тест:** Корабль садится на платформу, F → пешеход стоит рядом.

**Коммит:** ✅ → тест → следующий шаг

## 🎮 Управление (будущее)

| Клавиша | Действие |
|---------|----------|
| **F** | Переключение режимов (пеший ↔ корабль) |
| **W/S** | Вперёд/назад (оба режима) |
| **A/D** | Стрейф (пеший) / Крен (корабль) |
| **Мышь** | Вращение камеры (пеший) / Pitch+Yaw (корабль) |
| **Space** | Прыжок (пеший) |
| **Shift** | Бег (пеший) / Ускорение (корабль) |
