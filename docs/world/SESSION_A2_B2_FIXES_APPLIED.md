## Попытка 1: Исходный код (первая генерация)

**Формула:**
```csharp
height = peak.worldPosition.y; // 8848
radius = peak.baseRadius;       // 600
```

**Результат:** Горы-"иглы" — высота 8848 units, радиус 600, h/r = 14.7

**Причина:** Использовали worldPosition.y как высоту меша, но Y = метры, а меш должен быть в игровых units.

---

## Попытка 2: Масштаб высот /100

**Формула:**
```csharp
height = peak.realHeightMeters * 0.01f; // 8848 * 0.01 = 88.48
radius = peak.baseRadius;                // 600
```

**Результат:** Горы-"блинчики" — высота 88.48 units, радиус 600, h/r = 0.147

**Причина:** Слишком маленькая высота. realHeightMeters/100 даёт scaled units из старой документации, но радиус остался в "игровых units" — несовместимые масштабы.

## Попытка 3: Ghibli multiplier ×5

**Формула:**
```csharp
height = realHeightMeters * 0.01f * 5f; // 8848 * 0.01 * 5 = 442.4
radius = baseRadius;                     // 600
```

**Позиционирование:**
```csharp
transform.position.y = worldPosition.y - (meshHeight / 2);
// 8848 - 221.2 = 8626.8
```

**Результат:** Горы всё ещё мелкие и кучно.

**Гипотеза:** Проблема НЕ в формуле высоты, а в чём-то другом:
- Возможно X,Z координаты неправильные
- Возможно нужно другое значение heightScale
- Возможно проблема в визуальном восприятии (камера далеко)

## Гипотезы для отладки

### H1: Scene View vs Play Mode
- Scene View может показывать неправильно из-за far clip plane
- **Проверка:** Play mode, приблизить камеру к горе

### H2: Mesh не виден из-за масштаба
- Меш 442 units на позиции Y=8627 — камера по умолчанию на Y=10, далеко
- **Проверка:** Переместить камеру рядом с горой

### H3: X,Z координаты НЕ применяются
- Возможно worldPosition.x,z не используются при позиционировании
- **Проверка:** Debug.Log(transform.position) после BuildPeakMesh()

### H4: meshHeightMultiplier слишком маленький
- Текущий: 5.0, возможно нужно 50 или 100
- **Проверка:** Эксперимент в Inspector

### H5: baseRadius слишком большой
- Эверест baseRadius = 600, но meshHeight = 442
- h/r = 0.74 — нормально, но визуально может быть плоско
- **Проверка:** Уменьшить baseRadius до 200-300
