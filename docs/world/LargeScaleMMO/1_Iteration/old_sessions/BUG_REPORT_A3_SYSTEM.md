## Корневая проблема

### Проблема 1: Абсолютный масштаб

**Формулы дают ПРАВИЛЬНОЕ h/r, но НЕПРАВИЛЬНЫЙ абсолютный размер.**

```
Текущий: Эверест meshHeight = 450 units
Нужный:  Эверест meshHeight ≈ 600-800 units (визуально)
Множитель: ~1.5-2x от текущего
```

**НО:** пользователь поставил scale 60, что даёт:
```
450 * 60 = 27,000 units — это НЕ 600-800!
```

**Вывод:** scale 60 на родителе — НЕ решение, а маскировка проблемы.

### Проблема 2: Формы мешей

**CylinderMesh с radiusFactor Lerp(1, 0.15, pow(t, 0.5)):**
- t=0: radiusFactor = 1.0
- t=0.5: radiusFactor = 0.61
- t=1.0: radiusFactor = 0.15

Это даёт "пузатый" цилиндр с острой вершиной — "странный блин-пипка".

**ConeMesh с radiusFactor Lerp(1, 0.1, t*t):**
- t=0: radiusFactor = 1.0
- t=0.5: radiusFactor = 0.525
- t=1.0: radiusFactor = 0.1

Это даёт конус с резкой вершиной — "пипка".

**Пользователь хочет:** простой конус, НЕ "пипку".

### Проблема 3: CapsuleCollider

CapsuleCollider — это капсула (цилиндр + 2 полусферы). Для горы-конуса:
- CapsuleCollider выглядит как "толстая сфера"
- НЕ повторяет форму конуса
- Корабль сталкивается НЕ с горой, а с "пузырём" вокруг неё

**Вывод:** scale 60 — НЕ прямой множитель. Нужно подобрать empirically.

### 2. Формы мешей

**УБРАТЬ:** Tectonic, Volcanic, Dome, Isolated
**ОСТАВИТЬ:** ТОЛЬКО Cone

**Cone:**
- Простая геометрия
- Предсказуемый результат
- Легко настраивать

```csharp
// Простой конус:
for (ring = 0 to rings):
    t = ring / rings
    radius = baseRadius * (1 - t)  // Линейное сужение
    y = t * meshHeight
```

### 3. Коллайдеры

**Вариант A: MeshCollider**

meshCollider.sharedMesh = mesh;
meshCollider.convex = true;

**Вариант B: Несколько CapsuleCollider**

**Рекомендация:** MeshCollider с convex = true
