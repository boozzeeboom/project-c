## Что НЕ работает (переходит в сессию 5_3)

### Критическое: Между модули не работают как задумано

**Текущее поведение:**
- LeftShift + W/S активирует между pitch, но эффект кратковременный
- Частицы появляются только при активации
- Управление через LeftShift неудобно

**Ожидаемое поведение (требование пользователя):**
- Модуль установлен → между активен ПОСТОЯННО (без расхода топлива)
- Нажатие Z → нос корабля уходит вниз, начинает расходовать топливо, работают частицы
- Нажатие C → нос корабля уходит вверх, расход топлива, частицы
- Аналогично для других модулей: модуль даёт пассивный эффект, а направления клавиш дают активный выхлоп с расходом
- Перегрев после 10 сек непрерывного использования

**Корневая проблема:**
- Архитектура модулей не соответствует задумке. Нужно: пассивный режим (без расхода) + активный режим (направление + расход + частицы)
- Текущая реализация требует нажатия LeftShift + направление, что неудобно
- Нужна другая модель: модуль установлен = пассивный эффект, клавиша направления = активный выхлоп

### Нерешённое: Модули ROLL и YAW "not found on ship"

**Симптомы:**
- MODULE_MEZIY_PITCH работает
- MODULE_MEZIY_ROLL и MODULE_MEZIY_YAW выдают "not found on ship"
- Все 3 модуля установлены в ModuleSlot (подтверждено скриншотом Inspector)

**Диагностика:**
- `ShipModuleManager.Initialize()` теперь принудительно пересинхронизирует слоты
- Добавлено логирование содержимого каждого слота
- Нужно посмотреть логи при следующем запуске
