# Lore — пеший vs корабельный бой в мире Project C

> **Дата:** 2026-06-25 (v0.3)
> **Цель:** зафиксировать лор-факты, на которых базируется combat-engine. Только подтверждённое в `docs/COLABORATION.md`, `docs/ART_BIBLE.md`, `docs/gdd/GDD_01..02`, `docs/gdd/GDD_10_Ship_System.md`.
> **Ключевая идея:** combat-engine — **anti-restrictive**. Движок не знает, что есть «только пешие». Лор обосновывает, что **в будущем** будет и пеший, и корабельный, и оба используют одну ERPR-формулу.

---

## 1. Базовые принципы мира (подтверждено)

Из `docs/COLABORATION.md` (line 9-26) + `docs/ART_BIBLE.md` (line 11-26):

- **Sci-Fi + Ghibli** — сталь, ржавчина, кирпич + мягкие формы.
- **Нет магии** — антигравий, мезий, генераторы. *«Даже свечение — это физика, не волшебство»*.
- **Мир над облаками** — небо, ветер, корабли, коридоры высот (1200-4450м).
- **2090 год**, 5 Гильдий, Новое Правительство, подпольные организации.

**Вердикт для combat:** без магии. **Физика + технологии**. ERPR-формула — физика (dice + crit + defense). Антигравий = манипуляция g-полем. Мезий = опасный газ.

## 2. Пеший бой (MVP) — подтверждено в лоре

### 2.1 Оружие холодное (массовое)

Из `Battle/02_LORE.md §2.1`: порох дефицитен → холодное оружие = база. Мечи, копья, булавы, кинжалы, древковое.

**Следствие для движка:** Melee-ветка = `IWeapon` с `Range = 1-3м`, `DamageDice = d4-d10`. **Без магии**.

### 2.2 Стрелковое: арбалеты + пневматика + мезиевое

Из `Battle/02_LORE.md §2.2-2.3`:
- **Арбалеты** — массовое, механическое.
- **Пневматика** — сжатый воздух.
- **Мезиевое** — продвинутое, опасное.

**Следствие:** Ranged-ветка = `IWeapon` с `Range = 5-100м`, `DamageDice = d8-d12`. **Без пороха** (если game-designer не подтвердит, см. `Battle/02_LORE.md §6.5`).

### 2.3 Антигравийное оружие (уникально для Project C)

Из `Battle/02_LORE.md §2.4`: антигравий = манипуляция g-полем. Холодное оружие, мины, гранаты.

**Следствие:** Antigrav-ветка = `IWeapon` с `DamageType = Antigrav`, `critMod = +10` (g-поле «притягивает» к уязвимости). **Без магии** — это физика.

### 2.4 Броня

Из `Battle/02_LORE.md §3`: clothing (wearable, видно) + module (импланты, видно). `armorDefense` = физическая защита.

**Следствие:** `IDamageTarget.GetArmorDefense()` = sum экипированной одежды по armor-slots.

### 2.5 Что НЕ подтверждено в лоре для пешего боя

| # | Вопрос | Статус |
|---|---|---|
| L1 | Конкретные классы холодного оружия (меч vs сабля vs копьё) | Не подтверждено (placeholder-классы используются) |
| L2 | Дальность арбалета vs пневматики | Не подтверждено (default = 30м/50м) |
| L3 | Скорость перезарядки | Не подтверждено (default = weapon.range / 2) |
| L4 | Пороховое оружие (есть или нет) | Не подтверждено (предполагаем: нет, порох дефицитен) |
| L5 | NPC-враги (5 Гильдий, бандиты, пираты) | Частично подтверждено (5 Гильдий), но NPC-конфиги не описаны |
| L6 | Антиграв-щит | Подтверждено пользователем (2.4 = «есть»). Дизайн в `Battle/20_SKILL_TREES.md §5.1`. |

**Вердикт:** для движка MVP — **placeholder-значения** (default 30м/50м, default NPC как Goblin), designer-конфигурируемые.

## 3. Корабельный бой (FUTURE, Phase 3) — зафиксировано в GDD 10

> **Важно:** это **future**, не MVP. Движок проектируется **extensible** под ship combat **с самого начала** (anti-restrictive), но реализация — в `T-RTC16..T-RTC20` (Phase 3).

### 3.1 Корабли как «воздушные баржи» (GDD 10 §1.1)

Из `docs/gdd/GDD_10_Ship_System.md`:
- **Не истребители** — медленные, текучие, с инерцией.
- **Коридоры высот** — 1200-4450м (вне — Завеса).
- **Co-Op-first** — несколько пилотов = распределённое управление.
- **Модули** изменяют поведение (адаптивность).

**Следствие для движка:** `ShipAttacker` adapter использует `ShipController._rb.position` (GDD 10 уже реализован в `Player/ShipController.cs:352-355`). **Без рефакторинга** `ShipController.cs`.

### 3.2 Модули корабля (FUTURE)

Из GDD 10: модули = sensor, processor, weapon, utility. Уже частично реализовано (см. `Battle/01_ANALYSIS.md §1.3`).

**Следствие для ship combat:**
- `TurretDamageSource : IDamageSource` — оружейный модуль.
- `SensorDamageSource : IDamageSource` — датчик (узнавание, без урона).
- Корабль имеет **массив** модулей, каждый = `IDamageSource`.

### 3.3 Корабельный бой — что это (FUTURE)

> **Это не сейчас**, но **архитектура должна позволять**.

Гипотеза (не подтверждено в лоре, открыто для game-designer'а):
- Корабли сближаются, турели стреляют.
- Типы турелей: кинетические (баллистика), антиграв-импульсные (g-волна), мезиевые (токсин).
- Броня корабля = armorHull + armorShield.
- **Та же ERPR-формула** — но damage dice = d20 (турель стреляет болтами/снарядами крупнее).

**Вердикт:** для движка — **не** проектируем конкретные классы турелей. Создаём **hooks** (`IDamageSource` interface), которые позволяют реализовать турели позже.

### 3.4 Что НЕ подтверждено в лоре для ship combat

| # | Вопрос | Статус |
|---|---|---|
| L7 | Корабельное оружие (турели, ракеты) | Не подтверждено (GDD 10 не описывает) |
| L8 | Корабельный damage-type (отдельный от пешего?) | Не подтверждено (предполагаем: тот же Physical/Ballistic/Antigrav/Explosive) |
| L9 | PvP-дуэль на кораблях (1v1, 2v2) | Не подтверждено (открыто для game-designer'а) |
| L10 | NPC-враждебные корабли (пираты, разбойники) | Не подтверждено (5 Гильдий — мирные?) |
| L11 | Корабли разных размеров (от шлюпки до линкора) | Не подтверждено (GDD 10 говорит «баржи») |

**Вердикт:** для движка — **placeholder-корабли** (один тип), designer-конфигурируемые.

## 4. Общая лор-база для combat (пеший + ship)

### 4.1 Damage-типы (5 типов)

Из `Battle/02_LORE.md §4.1` + `Battle/10_DESIGN.md §3.1`:

| Тип | Источник | Лор-обоснование |
|---|---|---|
| **Physical** | холодное оружие | массовое, без фантастики |
| **Ballistic** | арбалет, пневматика | снаряды с кинетической энергией |
| **Antigrav** | антиграв-оружие, мины, гранаты | манипуляция g-полем (см. `ART_BIBLE.md`: голубое свечение) |
| **Explosive** | гранаты, мины, заряды | взрывная волна, осколки |
| **Mesium** | утечка мезия, повреждённый баллон | опасный газ (см. `ART_BIBLE.md`: зелёное свечение) |

**В движке:** `enum DamageType` (5 значений). **Все** типы — технологичные, **ни одного** магического.

### 4.2 Палитра свечений (для VFX)

Из `docs/ART_BIBLE.md` (line 73-76):
- **Голубое** = антигравий (безопасно)
- **Зелёное** = мезий (опасно)
- **Фиолетовое** = Завеса (смерть)

**Следствие для combat VFX:**
- Антиграв-атака → голубой ореол.
- Мезиевая атака → зелёный ореол + warning-sound.
- Попадание в Завесу → фиолетовая вспышка (world hazard, не combat).

### 4.3 Броня vs Защита (физика)

Из `Battle/02_LORE.md §4.1`:
- `armorDefense × typeMultiplier` — Physical/Ballistic: ×1.0, Antigrav: ×0.5, Explosive: ×0.7, Mesium: ×0.0.
- **Броня частично игнорируется g-волной** (антиграв пробивает).
- **Мезий-токсин не блокируется бронёй** (нужна герметичная защита).

**В движке:** это **одна формула** для пешего и корабельного. У корабля `armorDefense` больше (десятки/сотни), но **та же механика** (Physical: ×1.0, Antigrav: ×0.5).

### 4.4 Что НЕ делаем (lore-prohibited)

- ❌ Магия (стихийная, школярская, любая).
- ❌ Урон по уязвимости стихий (fire/ice/lightning).
- ❌ Мистические артефакты с «душой».
- ❌ Невидимость, телепортация, mind control.
- ❌ Лечение через молитву (только технологии/медикаменты).

| Сценарий | Участники | Damage-типы | Дистанция |

| **PvE-данж** (отдельный документ `turn-based-battles/`, отложен) | 1 Player + N NPC | все 5 типов | (TB-механика) |

### 5.2 Корабельный (FUTURE, Phase 3)

| Сценарий | Участники | Damage-типы | Дистанция |
|---|---|---|---|
| **Корабль vs NPC-корабль** | ShipAttacker + NpcShipAttacker | Ballistic, Antigrav, Mesium | 100-1000м |
| **PvP-корабль vs корабль** | ShipAttacker + ShipAttacker | все 5 типов | 100-1000м |

|---|---|

- ❌ NPC-враги в open world (HostileNPC, FactionAI) — **отдельная подсистема**.
- ❌ VFX, анимации, sound — 3D/audio отделы.
- ❌ UI (damage numbers, hit feedback) — `T-RTC10` (Phase 2, MVP без него).
- ❌ Корабельный damage-type (отдельный от пешего или тот же) — открыто для game-designer'а.
- ❌ PvP-фракции (5 Гильдий, враждебные игроки) — future.
- ❌ Replay-система, voice-chat — future.
- ❌ Серверные тики для NPC-AI — `ServerFixedUpdate` в `ShipController` (уже есть).

## 7. Резюме для дизайна движка

| Аспект | Пеший (MVP) | Ship combat (FUTURE) |
|---|---|---|
| **Damage-формула** | ERPR (готова в `Battle/10_DESIGN.md §7`) | ТА ЖЕ (через `IDamageSource` interface) |
| **Damage-типы** | 5 (Physical, Ballistic, Antigrav, Explosive, Mesium) | ТЕ ЖЕ 5 (или отдельный enum — открыто) |
| **HitLocation** | `locMult = 1.0` (отключён) | `locMult = 1.0` (отключён, см. 2.17) |
| **Defense (armor)** | `armorDefense × typeMultiplier` | ТА ЖЕ |
| **SkillMult** | Без cap (см. 2.18) | ТА ЖЕ |
| **Crit** | `1d100 + critMod >= 100` → ×2 | ТА ЖЕ |
| **Range** | 0-100м (меч-мезиевое) | 100-1000м (турель-корабль) |
| **Инициатива** | нет (real-time) | нет (real-time) |
| **Очередь ходов** | нет (real-time) | нет (real-time) |

**Главный вывод:** **ERPR-формула = универсальна** для пешего и корабельного боя. Движок оперирует **damage dice** + **crit** + **defense** + **skillMult** — без знания, **что** атакует (Player/Npc/Ship).

## 8. Открытые лор-вопросы (для game-designer'а)

### 8.1 Корабельный damage-type

**Вопрос:** для ship combat использовать **те же 5 damage-типов** (Physical/Ballistic/Antigrav/Explosive/Mesium) или **отдельный enum** (например, `ShipDamageType.HullPiercing / Explosive / Energy`)?

**Гипотеза:** те же 5 типов. **Лор-обоснование:** та же физика (g-волна, взрыв, мезий-токсин) действует и на корабль.

### 8.2 Корабельное оружие (турели)

**Вопрос:** турели на корабле — **отдельный подкласс** `IDamageSource` или **переиспользуют** `WeaponDamageSource` (как обычное оружие, но с `Range = 500м`)?

**Гипотеза:** `TurretDamageSource : IDamageSource` — отдельный класс с `DamageType = Ballistic`, `DamageDice = d20` (крупнее пешего), `Cooldown` (между выстрелами).

### 8.3 NPC-враждебные корабли

**Вопрос:** будут ли NPC-корабли, которые атакуют игрока? Сейчас `NpcShipController` мирный.

**Гипотеза:** да, враждебные NPC-корабли (пираты, разбойники) — отдельная подсистема, `NpcShipAttacker : NetworkBehaviour, IAttacker` (Phase 3).

### 8.4 ПвП-дуэли на кораблях

**Вопрос:** PvP только пеший, или ещё и корабельный?

**Гипотеза:** **оба**. Пеший — `T-RTC11..T-RTC15` (Phase 2). Корабельный — `T-RTC21+` (Phase 3+).

### 8.5 Корабельная броня vs щит

**Вопрос:** `armorDefense` для корабля = `armorHull` + `armorShield` (два слоя)? Или один?

**Гипотеза:** `armorHull` (базовая броня) + `armorShield` (энергетический щит, поглощает часть урона). Phase 3.

### 8.6 Что делает `ShipAttacker` если в корабле несколько пилотов

**Вопрос:** GDD 10 говорит «Co-Op-first» — несколько пилотов. Как combat-движок учитывает?

**Гипотеза:** каждый пилот = `IAttacker` (registred отдельно). Damage от турели = среднее damage пилотов × skill bonus. Phase 3+.
