```
inventory v2 bugfixes: scene-spawn BootstrapScene + pickup v2 path + P-tab lazy-subscribe

=== Phase 8: bugfixes после первой попытки тестирования ===

Три бага найдены, все исправлены. Документация в
docs/Character-menu/sub_inventory-tab/60_KNOWN_ISSUES.md §11.

--- BUGFIX 1: [InventoryServer] не заспавнен ---
Симптом: chest pickup → нет snapshot → TAB-колесо пустое.
Root cause: scene-placed NetworkObject в BootstrapScene имеют
InScenePlacedSourceGlobalObjectIdHash=0, NGO их не спавнит;
ScenePlacedObjectSpawner фильтровал только WorldScene_* и
пропускал BootstrapScene.
Fix: ScenePlacedObjectSpawner.SpawnInAllLoadedScenes() — убрал
фильтр, для не-WorldScene сцен спавним все scene-placed NetworkObject
вручную.
Verify: spawned=12, already=0, failed=0 в Scene(0,0).

--- BUGFIX 2: pickup не добавлял в инвентарь ---
Симптом: "предметы визуально подбираются, но в инвентаре не числятся".
Root cause: NetworkPlayer.TryPickup() вызывал _inventory.AddItem,
но _inventory == null после Phase 4 (SpawnInventory заноплен).
Предмет деактивировался через RPC, но никогда не попадал в
InventoryClientState.
Fix: fallback на v2: if (_inventory != null) _inventory.AddItem()
else _nearestPickup.Collect() (шлёт RequestPickup через v2 RPC).
Verify: PickupItem успешно подобран + InventoryWorld picked up ID=7.

--- BUGFIX 3: P-таб пуст при первом клике (нужен toggle через Рынок) ---
Симптом: P → Инвентарь (1-й клик) — пусто. P → Рынок → Инвентарь
(3-й клик) — работает.
Root cause (двухслойный):
  a) Lazy-subscribe race: в EnsureBuilt (OnEnable) InventoryClientState.Instance
     мог быть null (NetworkManagerController.Awake ещё не создал root GO).
     Подписка OnSnapshotUpdated += HandleInventorySnapshotUpdated
     пропускалась. Доказательство: в логе handlers=1 (только TAB-UI),
     CharacterWindow не подписан.
  b) ListView layout pitfall: первый display: none → flex на inventory-section
     не вызывал повторный layout — VE не создавались, bindItem не вызывался.
Fix (3 части):
  1) Lazy-subscribe в Update() CharacterWindow с флагом _isInventorySubscribed.
     SubscribeInventory() / UnsubscribeInventory() идемпотентны.
  2) RefreshInventoryCache + ApplyInventoryFilters: Rebuild() → RefreshItems()
     + null-trick (не переприсваиваем itemsSource если та же ссылка).
  3) SwitchTab("inventory"): _inventoryList.MarkDirtyRepaint() после
     display: flex.
Verify: handlers=2 (TAB-UI + CharacterWindow), HandleInventorySnapshotUpdated
срабатывает, P → Инвентарь сразу показывает содержимое.

--- Compile fix ---
Опечатка: добавил второй Update() — CS0111. Встроил lazy-subscribe
в существующий Update (Esc-handler).
```
