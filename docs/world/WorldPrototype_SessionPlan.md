## ⚠️ КРИТИЧНО: ПРАВИЛО МАСШТАБА СКРИПТОВЫХ ОБЪЕКТОВ

> **Всё, что создаётся программно через скрипты (FarmPlatform, TerraceBuilder, FerryCableSystem, MountainMeshBuilder и т.д.) — по умолчанию создаётся В 5-10 РАЗ МЕНЬШЕ нужного.**

**Опыт сессии A5 (FarmPlatform):**
- Платформа 40×20 → здания 6×4×4 → террасы 40×20 → всё «сбилось в кучу»
- При ручном увеличении ×5 фермы стали выглядеть нормально
- Координаты расположения правильные, проблема ТОЛЬКО в размере объектов

**Причина:** «Удобные» числа (40, 20, 6) слишком малы для мира, где горы — 300-600 units в радиусе.

**Решение:** Все размеры скриптовых объектов умножать ×5:

| Объект | Было | Нужно |
|--------|------|-------|
| Антигравийная плита | 40×20×2 | **200×100×10** |
| Здание | 6×4×4 | **30×20×20** |
| Терраса (1-я) | 40×1.5×20 | **200×7.5×100** |
| Террасы отступ | 5 | **25** |
| Вагонетка | 2 | **10** |
| Провисание троса | 20 | **100** |

**ВАЖНО:** Координаты (X,Y,Z) — ПРАВИЛЬНЫЕ, не менять! Масштабируются ТОЛЬКО размеры объектов.

## 6. Связь между системами Ландшафта и Облаков

### 6.1. Высотная связка

```
Завеса (Y=12) ───► Global Corridor min (Y=12) ───► Нижний слой облаков (Y=15-40)
                                                        │
                                                        ▼
Фермы (Y=25-45) ───► Middle слой облаков (Y=40-70) ───► Города (Y=48-88)
                                                        │
                                                        ▼
Кумуло-дождевые (Y=12-90) ───► Высотный коридор (Y=60+) ───► Снежные вершины (Y=70+)
```

### 6.2. Визуальная связка

| Элемент ландшафта | Влияние на облака |
|-------------------|-------------------|
| **Высота города** | Определяет min/max коридора -> плотность облаков в этом диапазоне |
| **Климат массива** | CloudClimateTinter -> цвет облаков меняется |
| **Горные хребты** | Создают «ветровую тень» — облака реже за хребтом |
| **Завеса** | Фиолетовый оттенок на нижних облаках near Y=12 |
| **Кумуло-дождевые** | Растут из завесы, привязаны к зонам высокой влажности (фермы near завесы) |

### 6.3. Геймплейная связка

| Геймплей | Ландшафт | Облака |
|----------|----------|--------|
| **Полёт между городами** | Расстояние определяет время полёта | Облака создают препятствия, турбулентность |
| **Навигация** | Маяки, хребты, силуэты пиков | Разрывы в облаках открывают ориентиры |
| **Торговля** | Фермы производят товары, города торгуют | Облака влияют на скорость и расход топлива |
| **Опасность** | Высота горы -> риск столкновения | Завеса -> смерть, кумуло-дождевые -> молнии |
| **Исследование** | Заброшенные платформы на вторичных пиках | Облака скрывают секретные локации |

## 9. Риски прототипа

| Риск | Вероятность | Влияние | Митигация |
|------|------------|--------|-----------|
| **Горы выглядят одинаково** | Средняя | Высокое | Ручные heightmap-профили для 5 главных пиков (уже описаны в TechnicalDesign) |
| **Завеса видна как плоскость** | Высокая | Среднее | Depth fade + horizon blending + curvature на краях |
| **Кумуло-дождевые перекрывают обзор** | Средняя | Среднее | Ограничить 3-5 штуками, разместить вдали от основных маршрутов |
| **FPS падает при полном мире** | Низкая | Высокое | LOD2 billboard + frustum culling + async loading |
| **Фермы «висят» без привязки к хребтам** | Средняя | Среднее | TerraceBuilder raycast к ridge mesh при размещении |
| **Городские префабы слишком однообразны** | Высокая | Среднее | 5 уникальных наборов (по одному на город), разная архитектура |
