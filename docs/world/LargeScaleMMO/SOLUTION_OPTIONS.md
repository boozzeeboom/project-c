# FloatingOrigin — Варианты решения

## Текущая проблема

Пользователь хочет: "игрок перемещается по открытому миру до 1М без артефактов, горы и облака остаются на своих местах"

## Вариант A: Floating Origin (текущий) ✅ Работает

**Принцип:** Мир уезжает от игрока, игрок остаётся рядом с origin

```
Игрок на 1М:
  Игрок: (1000000, 5, 0) — НЕ двигается
  TradeZones: (0, 0, 0) — привязан к игроку
  WorldRoot: (-1000000, 0, 0) — сдвинут
  
Визуально: горы уехали, игрок видит пустое пространство
```

**Реализация:** FloatingOriginMP как есть

---

## Вариант B: Floating Origin с движением мира К игроку

**Принцип:** WorldRoot движется навстречу игроку, TradeZones тоже движется

```
Игрок на 1М:
  Игрок: (1000000, 5, 0) — остаётся на месте
  TradeZones: (1000000, 0, 0) — движется с игроком
  WorldRoot: (0, 0, 0) — остаётся на месте

Визуально: горы на своих местах, игрок рядом с ними
```

**Проблема:** В MMO все игроки должны видеть одинаковый мир. Если каждый игрок двигает мир к себе — будет хаос!

---

## Вариант C: Chunk Streaming (правильный для MMO) ⭐ Рекомендуется

**Принцип:** Мир делится на чанки. Загружаются только чанки вокруг игрока.

```
Мир: 10,000 x 10,000 units
Чанк: 2,000 x 2,000 units

Игрок на позиции (5000, 0):
  Загружены чанки: [2,0], [2,1], [3,0], [3,1], [2,-1], [3,-1]
  Все остальные — выгружены

Визуально: игрок видит только локальную часть мира
Преимущество: не нужна работа с float > 100k
```

**Реализация:** WorldChunkManager + ChunkLoader

---

## Рекомендация

**Для MMO проекта:** Вариант C (Chunk Streaming)

- Игрок всегда в центре, мир вокруг него
- Нет проблем с float precision
- Сервер управляет загрузкой чанков
- Масштабируется на огромные миры

**Вариант A можно доработать:** сделать автоматический сдвиг когда игрок > 100k, чтобы он возвращался к origin.
