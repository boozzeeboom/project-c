## Боевой движок: за неделю с нуля

В начале недели боя в игре не было. В конце — 24 C#-файла, 5 формул урона,
Skill Tree с 27 навыками, AOE-система, прицеливание, анимации.

**Как работает.** Всё считает сервер. Игрок нажал кнопку → RPC на сервер →
сервер бросил d8 → посчитал хит/мисс/крит → применил броню → скормил множители
от выученных навыков → отправил результат всем. Клиент только показывает
красивые цифры. Читерство невозможно.

Формула урона вышла простой и прозрачной:
- d8 + атака оружия минус защита брони
- На 1 — промах, на 8 — крит (×1.5)
- У каждого типа урона (физический, магический, антиграв) свой множитель к броне

Потом пошли AOE-атаки. Пять форм зон поражения: сфера, конус, коробка, линия,
радиальная. На конусе я затупил с тригонометрией, пришлось перечитывать
школьную геометрию. Зато визуализация в редакторе работает — можно увидеть
красный сектор удара перед атакой.

## Skill Tree: 27 навыков, которые что-то дают

Дерево навыков — 27 узлов по 5 дисциплинам: Ближний бой, Дальний бой, Техники
(в мире нет магии, но мезиевые технологии работают как заклинания), Защита,
Поддержка.

Каждый навык — обычный ScriptableObject. Дизайнер может ткнуть пальцем:
стоимость в очках навыков, пререквизиты, множители к урону. Без программиста.

UI — граф с зумом и панорамой. Активные навыки подсвечиваются, пассивные
отдельным бейджем. Кнопка [ИЗУЧИТЬ НАВЫК] проходит серверную валидацию —
нельзя хакнуть клиент и выучить всё сразу.

Бонусом: анимации навыков. Нажал Q — аниматор подменил клип атаки на клик
конкретного навыка, AnimationEvent дёрнул RPC на сервер, сервер рассчитал урон.
Цепочка длинная, но работает без затыков.

## Кастомизация: один день на всё

В воскресенье сделал полный цикл кастомизации персонажа. Пол (M/F),
6 пресетов телосложения (от «хлюпик» до «шкаф»), цвет кожи, волос, одежды.
Причёска (пока две — лысый и короткий, остальное на следующей неделе).

UI переиспользовали от Skill Tree Window — свой паттерн уже есть,
набиваем руку.

**Баг дня.** Персонаж при загрузке становится невидимо маленьким и тёмно-серым.
Искал два часа. Причина — C# struct сбрасывается в default после domain reload
(рост = 0, ширина = 0). Фикс на пять строк в методе Awake. Тот случай, когда
язык программирования мстит за веру в дефолтные значения.

## NPC-враги: первые гоблины

До этой недели NPC только давали квесты. Теперь — агрессивные гоблины
с конечным автоматом:

- Стоят → заметили игрока → бегут → атакуют в упор → умирают → дропают лут
- Если игрок убежал далеко — возвращаются на точку спавна (30 метров leash)
- Спавн проверяет, что поверхность твёрдая (не в воздухе же)

У гоблина есть CharacterController, NavMeshAgent, NetworkObject, NpcAttacker,
NpcTarget и Animator с 5 состояниями. Много компонентов, но каждый делает
своё дело.

Анимаций пока базовые — заглушки, ждут нормальные Kevin Iglesias FBX.

## Input System: прощай, хардкод

Самое тоскливое за неделю — переписывать ввод. Раньше все 31 действие были
жёстко зашиты в Update: `Keyboard.current.wKey.isPressed`. Каждый раз
менять в коде, чтобы переназначить кнопку.

Теперь — единый ScriptableObject со всеми биндами. Можно переназначить любую
клавишу через Esc-меню. Сохраняется в PlayerPrefs (JSON). Есть кнопка
[СБРОСИТЬ НА СТАНДАРТНЫЕ].

Попутно пофиксил баг: CharacterWindow крал Esc и не давал открыть системное
меню. Теперь Esc сначала закрывает окно, потом — меню.

## Equipment Visual: как мы чуть не положили сервер RPC-шумом

Визуал экипировки — когда надел меч, он появился в руке персонажа. Звучит
просто. На деле: 10+ колбэков одновременно вызывали `RequestEquipRpc`,
сервер захлёбывался. Починил userData-based unregister — каждый колбэк
регистрируется один раз.

Bone mapping: 8 слотов (оружие, щит, шлем, броня, наручи, перчатки, сапоги,
пояс) привязаны к костям скелета через HumanBodyBones enum. Элегантно.

## Процесс: что пошло не так

1. **Забыл про domain reload.** Unity при перезагрузке домена сбрасывает
struct'ы в default. Известная грабля, но каждый раз наступаешь.

2. **Rate limit на RPC.** Десять колбэков → десять RPC → сервер в ступоре.
Починил, но мог предусмотреть раньше.

3. **Неделя без PlayMode-тестов.** Всё проверял руками. Классика. Знаю,
что надо, но когда каждая система пишется за день — времени нет.

4. **Распыление.** Combat, Customisation, NPC Enemy, Input System, Equipment
Visual — пять направлений за одну неделю. Каждое сделано, но чувствуется,
что глубже можно было копнуть по одному.
