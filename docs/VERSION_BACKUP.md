## 📦 Создание резервной копии

### Вариант 1: Git тег (рекомендуется)

```bash
# Создать тег текущей версии
git tag v0.3.0-2026-04-02

# Отправить тег на GitHub
git push upstream --tags
```

**Преимущества:**
- ✅ Быстрый доступ к версии
- ✅ Не занимает место
- ✅ Видно историю

---

### Вариант 2: Резервная ветка

```bash
# Создать резервную ветку
git checkout -b backup/2026-04-02

# Вернуться на рабочую
git checkout qwen-dev

# Отправить на GitHub
git push -u upstream backup/2026-04-02
```

**Преимущества:**
- ✅ Можно вернуться и продолжить с этой точки
- ✅ Видны изменения между версиями

---

### Вариант 3: Внешняя копия (полная)

```bash
# В проводнике Windows:
# 1. Скопируй папку C:\UNITY_PROJECTS\ProjectC_client
# 2. Вставь как C:\UNITY_PROJECTS\ProjectC_client_backup_2026-04-02
```

**Преимущества:**
- ✅ Полная копия включая Library/
- ✅ Быстрое восстановление
- **Недостатки:** Занимает ~5-10 ГБ

## 🔄 Восстановление версии

### Из Git тега

```bash

git tag --list

git checkout v0.3.0-2026-04-02

# Или создать ветку из тега
git checkout -b restore/v0.3.0 v0.3.0-2026-04-02
```

---

### Из резервной ветки

```bash

git branch -a

git checkout backup/2026-04-02
```

## 🎯 Инструкция для Qwen Code

### Перед началом работы:

```bash
# 1. Проверить версию
git status
git log --oneline -1

# 2. Создать резервную точку (если планируются крупные изменения)
git tag backup/pre-$(date +%Y-%m-%d)
```

### После работы:

```bash
# 1. Проверить изменения
git status

# 2. Закоммитить
git add .
git commit -m "Описание изменений"

# 3. Отправить на GitHub
git push upstream qwen-dev
```

## ⚠️ Аварийное восстановление

### Если всё сломалось:

```bash
# 1. Откатиться к последнему коммиту
git reset --hard HEAD

# 2. Или к последнему коммиту на upstream
git fetch upstream
git reset --hard upstream/qwen-dev

# 3. Очистить кэш Unity
# В проводнике: удали папку Library/
# Открой Unity заново
```
