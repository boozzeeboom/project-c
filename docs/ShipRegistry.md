**Последнее обновление:** Апрель 2026 | **Версия:** 1.0 | **Ветка:** `qwen-gamestudio-agent-dev`

Этот документ — **единый источник истины** для всех кораблей, модулей и их совместимости.

## Как Пользоваться Этим Документом

- Каждый корабль имеет уникальный **ShipID** (для кода)
- Каждый модуль имеет уникальный **ModuleID** (для кода)
- Матрица совместимости внизу показывает что можно ставить на что
- GDD_10_Ship_System.md содержит правила дизайна; это — каталог
- Добавляйте новые записи по шаблону

## 6. Система Ключ-Стержней

### 6.1 Как Работает
```
KeyRod (ScriptableObject):
├── keyRodId: "KR-0042"
├── registeredShipId: "SHIP_LG_02_0078"
├── ownerPlayerId: "player_abc123"
├── isDuplicate: false  (true = нелегальная копия)
└── accessLevel: Full   (Full | Limited | OneTime)
```

### 6.2 Поток
1. Игрок получает KeyRod при покупке/находке корабля
2. KeyRod хранится в инвентаре
3. Подход к кораблю → F → проверка KeyRod
4. Совпадение shipId → Engine On
5. Чужой KeyRod → можно угнать (но СОЛ уведомит владельца)

### 6.3 Угон
- Чужой KeyRod + запуск = угон
- Владелец: уведомление через CommPanel
- В зоне СОЛ: корабль помечается, возможна блокировка
- С `MODULE_STEALTH`: задержка обнаружения ×3

## 7. Pipeline Добавления Нового Корабля

```
1. Game Designer создаёт запись в этом документе (черновик)
2. Technical Director утверждает класс и статистику
```
