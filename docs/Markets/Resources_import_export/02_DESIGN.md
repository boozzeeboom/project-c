## TL;DR

Один CSV-файл `Resources_Import.csv` с **5 секциями** (inventory / tradeItems / marketItems /
exchangeRates / recipes), разделёнными пустой строкой и строкой-маркером `# block=...`. Editor
Window `Tools → ProjectC → Resources → CSV Import/Export` парсит CSV, показывает Preview
(кол-во строк, ошибки по строкам, глобальные ошибки), на импорте создаёт/обновляет SO во всех
5 подсистемах, пересобирает `ItemRegistry` и `ExchangeRateConfig`. Экспорт — обратная
операция: сканирует все ассеты → генерирует CSV. Все ID через **itemName** (для inventory)
и **itemId** (для trade), cross-validate до импорта.

## 1. Формат CSV

### 1.1 Общие правила

- **Кодировка:** UTF-8 with BOM (для Excel/LibreOffice).
- **Перевод строк:** `\n` или `\r\n` (парсер принимает оба).
- **Quoted/escaped:** `"`, `""`, multiline — поддержка как в `QuestCsvParser` (копируем
  `SplitCsvLines` + `ParseCsvLine`).
- **Header в каждой секции** — обязателен, canonical names + aliases (как у квестов).
- **Column aliases** — русские варианты: "Название", "Тип", "Описание" и т.д.
- **Default values** — заданы в `ColumnDef.defaultValue`, применяются если ячейка пуста.
- **Пустые строки** между секциями — игнорируются, кроме случая, когда после пустой строки
  идёт строка `# block=...` (это маркер начала новой секции).

## 3. Алгоритм импорта

### 3.1 Этапы

```
1. SELECT CSV file (user) → path
2. PARSE CSV into 5 блоков (split by "# block=...")
3. VALIDATE per-block (column types, required, duplicates)
4. CROSS-VALIDATE (itemName ↔ tradeItemId)
5. PREVIEW (in Window: rows count, errors, warnings)
6. USER confirms → APPLY
7. CREATE/UPDATE each SO per block
8. UPDATE ItemRegistry, ExchangeRateConfig, TradeItemDatabase, MarketConfig
9. SaveAssetDatabase + Refresh
10. REPORT (created/updated/skipped/errors, dialog)
```

### 3.2 Шаг 2 — Парсинг CSV (детально)

```csharp
public class ResourcesCsvParser
{
    public static (Dictionary<string, List<ResourcesCsvRow>> blocks, List<string> errors) ParseFile(string path)
    {
        var text = File.ReadAllText(path, Encoding.UTF8);
        var lines = ResourcesCsvParser.SplitCsvLines(text);  // копия из QuestCsvParser
        
        var blocks = new Dictionary<string, List<ResourcesCsvRow>>();
        var errors = new List<string>();
        string currentBlock = null;
        int blockLine = 0;
        string[] header = null;
        var columnMap = new Dictionary<string, int>();
        
        for (int i = 0; i < lines.Count; i++)
        {
            var line = lines[i].Trim();
            if (string.IsNullOrEmpty(line)) continue;
            
            // Block marker
            if (line.StartsWith("# block="))
            {
                currentBlock = line.Substring("# block=".Length).Trim();
                blocks[currentBlock] = new List<ResourcesCsvRow>();
                blockLine = 0;
                header = null;
                continue;
            }
            
            // Comments (lines starting with #) — ignore
            if (line.StartsWith("#")) continue;
            
            if (currentBlock == null)
            {
                errors.Add($"Line {i+1}: data before first '# block=' marker. Ignored.");
                continue;
            }
            
            var fields = ResourcesCsvParser.ParseCsvLine(line);
            
            // Header
            if (blockLine == 0)
            {
                header = fields;
                // Resolve canonical names
                var schema = ResourcesCsvSchema.GetBlockSchema(currentBlock);
                for (int j = 0; j < header.Length; j++)
                {
                    var canonical = schema.ResolveColumnName(header[j]);
                    if (canonical == null)
                    {
                        errors.Add($"Line {i+1} (header of '{currentBlock}'): unknown column '{header[j]}'. Ignored.");
                    }
                    else
                    {
                        columnMap[canonical] = j;
                    }
                }
                // Check required
                foreach (var col in schema.Columns.Where(c => c.required))
                {
                    if (!columnMap.ContainsKey(col.name))
                        errors.Add($"Block '{currentBlock}': missing required column '{col.name}'");
                }
            }
            else
            {
                // Data row
                var row = new ResourcesCsvRow { lineNumber = i+1, block = currentBlock };
                foreach (var kv in columnMap)
                {
                    if (kv.Value < fields.Length)
                        row.values[kv.Key] = fields[kv.Value].Trim();
                }
                ApplyDefaults(row, ResourcesCsvSchema.GetBlockSchema(currentBlock));
                ValidateRow(row);
                blocks[currentBlock].Add(row);
            }
            
            blockLine++;
        }
        
        return (blocks, errors);
    }
    
    // SplitCsvLines, ParseCsvLine — копия из QuestCsvParser (общий utility)
}
```

### 3.3 Шаг 3 — Per-block validation (типы, required, ranges)

```csharp
private static void ValidateRow(ResourcesCsvRow row)
{
    var schema = ResourcesCsvSchema.GetBlockSchema(row.block);
    
    // Required fields
    foreach (var col in schema.Columns.Where(c => c.required))
    {
        if (string.IsNullOrWhiteSpace(row.Get(col.name)))
            row.errors.Add($"'{col.name}' is required");
    }
    
    // Type checks
    foreach (var col in schema.Columns)
    {
        var v = row.Get(col.name);
        if (string.IsNullOrEmpty(v)) continue;
        switch (col.type)
        {
            case "int":
                if (!int.TryParse(v, out _))
                    row.errors.Add($"'{col.name}' = '{v}' is not int");
                break;
            case "float":
                if (!float.TryParse(v, NumberStyles.Float, CultureInfo.InvariantCulture, out _))
                    row.errors.Add($"'{col.name}' = '{v}' is not float");
                break;
            case "bool":
                if (!IsBool(v))
                    row.errors.Add($"'{col.name}' = '{v}' is not y/n/yes/no/true/false/1/0");
                break;
            case "enum":
                if (!Enum.TryParse(col.enumType, v, true, out _))
                    row.errors.Add($"'{col.name}' = '{v}' is not valid {col.enumType.Name}");
                break;
        }
    }
    
    // Range checks (specific to column)
    // maxStack >= 1, weightKg >= 0, basePrice >= 0, etc.
}
```

### 3.4 Шаг 4 — Cross-validation (между блоками)

```csharp
public static class ResourcesCsvCrossValidator
{
    public static void Validate(
        Dictionary<string, List<ResourcesCsvRow>> blocks,
        List<string> globalErrors)
    {
        var invNames = new HashSet<string>(StringComparer.OrdinalIgnoreCase);
        var tradeIds = new HashSet<string>(StringComparer.OrdinalIgnoreCase);
        var marketItems = new HashSet<(string, string)>();
        var exchangeRates = new HashSet<(string, string)>();
        
        // 1. Build indices
        if (blocks.TryGetValue("inventory", out var inv))
            foreach (var r in inv) if (!r.HasError) invNames.Add(r.Get("itemName"));
        
        if (blocks.TryGetValue("tradeItems", out var trd))
            foreach (var r in trd) if (!r.HasError) tradeIds.Add(r.Get("tradeItemId"));
        
        if (blocks.TryGetValue("marketItems", out var mkt))
            foreach (var r in mkt) if (!r.HasError)
                marketItems.Add((r.Get("tradeItemId"), r.Get("locationId")));
        
        if (blocks.TryGetValue("exchangeRates", out var xch))
            foreach (var r in xch) if (!r.HasError)
                exchangeRates.Add((r.Get("tradeItemId"), r.Get("inventoryItemName")));

        // Phase 2: if (blocks.TryGetValue("recipes", out var rec)) ... — CraftingCsvValidator

        // 2. inventory: uniqueness
        var dupInv = inv.GroupBy(r => r.Get("itemName"), StringComparer.OrdinalIgnoreCase)
            .Where(g => g.Count() > 1).Select(g => g.Key);
        foreach (var d in dupInv)
            globalErrors.Add($"inventory: duplicate itemName '{d}'");
        
        // 3. tradeItems: uniqueness
        var dupTrd = trd.GroupBy(r => r.Get("tradeItemId"), StringComparer.OrdinalIgnoreCase)
            .Where(g => g.Count() > 1).Select(g => g.Key);
        foreach (var d in dupTrd)
            globalErrors.Add($"tradeItems: duplicate tradeItemId '{d}'");
        
        // 4. marketItems: tradeItemId exists
        foreach (var r in mkt)
        {
            if (!r.HasError && !tradeIds.Contains(r.Get("tradeItemId")))
                r.errors.Add($"marketItems: tradeItemId '{r.Get("tradeItemId")}' not in tradeItems block");
        }
        
        // 5. exchangeRates: tradeItemId + inventoryItemName both exist
        foreach (var r in xch)
        {
            if (!r.HasError)
            {
                if (!tradeIds.Contains(r.Get("tradeItemId")))
                    r.errors.Add($"exchangeRates: tradeItemId '{r.Get("tradeItemId")}' not in tradeItems");
                if (!invNames.Contains(r.Get("inventoryItemName")))
                    r.errors.Add($"exchangeRates: inventoryItemName '{r.Get("inventoryItemName")}' not in inventory");
            }
        }
        
        // 6. recipes — Phase 2 (out of scope MVP).
        //    Парсер распознаёт секцию, но cross-validate не проверяет ингредиенты/outputs
        //    (это будет делать CraftingCsvValidator в Phase 2). Здесь только убеждаемся,
        //    что секция существует, иначе — global warning.
        if (blocks.ContainsKey("recipes"))
        {
            // Ничего не валидируем в Phase 1 — Phase 2 CraftingCsvValidator займётся.
            // Просто фиксируем факт для UI.
        }
    }
}
```

### 3.5 Шаг 6-7 — Apply (создание/обновление SO)

```csharp
public class ResourcesCsvImporter
{
    public class ImportResult
    {
        public int created, updated, skipped;
        public List<string> errors = new();
        public List<string> warnings = new();
        public int TotalChanges => created + updated;
    }
    
    public static ImportResult Apply(Dictionary<string, List<ResourcesCsvRow>> blocks)
    {
        var result = new ImportResult();
        
        // 1. Process inventory → ItemData + ItemRegistry
        ProcessInventory(blocks.GetValueOrDefault("inventory", new()), result);
        
        // 2. Process tradeItems → TradeItemDefinition + TradeItemDatabase
        ProcessTradeItems(blocks.GetValueOrDefault("tradeItems", new()), result);
        
        // 3. Process marketItems → MarketItemConfig (per locationId)
        ProcessMarketItems(blocks.GetValueOrDefault("marketItems", new()), result);
        
        // 4. Process exchangeRates → ExchangeRateEntry
        ProcessExchangeRates(blocks.GetValueOrDefault("exchangeRates", new()), result);
        
        // 5. recipes — Phase 2 (out of scope MVP)
        if (blocks.TryGetValue("recipes", out var recipes) && recipes.Count > 0)
            result.warnings.Add($"recipes: {recipes.Count} rows skipped (Phase 2, see Crafting roadmap)");
        
        // 6. Save
        AssetDatabase.SaveAssets();
        AssetDatabase.Refresh();
        
        return result;
    }
    
    private static void ProcessInventory(List<ResourcesCsvRow> rows, ImportResult result)
    {
        const string ITEMS_FOLDER = "Assets/_Project/Resources/Items";
        EnsureFolder(ITEMS_FOLDER);
        
        // Load existing registry (don't recreate)
        var registryPath = "Assets/_Project/Items/Data/ItemRegistry.asset";
        var registry = AssetDatabase.LoadAssetAtPath<ItemRegistry>(registryPath);
        if (registry == null)
        {
            result.errors.Add($"ItemRegistry not found at {registryPath}");
            return;
        }
        
        var registryEntries = registry.entries?.ToList() ?? new List<ItemRegistry.Entry>();
        int maxId = registryEntries.Count > 0 ? registryEntries.Max(e => e.id) : 0;
        
        foreach (var row in rows)
        {
            if (row.HasError) { result.skipped++; continue; }
            
            var name = row.Get("itemName");
            var typeStr = row.Get("itemType");
            if (!Enum.TryParse<ItemType>(typeStr, true, out var type))
            {
                row.errors.Add($"Unknown itemType '{typeStr}'");
                result.skipped++;
                continue;
            }
            
            // Find existing ItemData
            var existing = FindItemDataByName(name);
            
            if (existing == null)
            {
                // CREATE
                var item = ScriptableObject.CreateInstance<ItemData>();
                item.itemName = name;
                item.itemType = type;
                item.description = row.Get("description");
                item.icon = existing?.icon;  // preserve if update; null for new
                item.maxStack = row.GetInt("maxStack", 1);
                item.weightKg = row.GetFloat("weightKg", 0.1f);
                
                var path = $"{ITEMS_FOLDER}/Item_{type}_{SanitizeFileName(name)}.asset";
                EnsureUniqueAssetPath(ref path);
                AssetDatabase.CreateAsset(item, path);
                
                // Register in ItemRegistry
                int newId = ++maxId;
                if (registryEntries.Any(e => e.id == newId))
                    result.warnings.Add($"ItemRegistry: id {newId} already exists, skipping (item '{name}')");
                else
                {
                    registryEntries.Add(new ItemRegistry.Entry { id = newId, item = item });
                    result.created++;
                }
            }
            else
            {
                // UPDATE
                existing.itemName = name;  // potentially rename
                existing.itemType = type;
                existing.description = row.Get("description");
                // DO NOT touch existing.icon (CSV doesn't carry Sprite refs)
                existing.maxStack = row.GetInt("maxStack", existing.maxStack);
                existing.weightKg = row.GetFloat("weightKg", existing.weightKg);
                EditorUtility.SetDirty(existing);
                result.updated++;
            }
        }
        
        // Save registry
        registry.entries = registryEntries.ToArray();
        EditorUtility.SetDirty(registry);
    }
    
    // ProcessTradeItems, ProcessMarketItems, ProcessExchangeRates — аналогично
    // ProcessRecipes — Phase 2 (out of scope MVP)
}
```

## 4. Алгоритм экспорта

Обратная операция: сканируем все ассеты → генерируем CSV.

```csharp
public static class ResourcesCsvExporter
{
    public static int Export(string csvPath)
    {
        var sb = new StringBuilder();
        
        sb.AppendLine("# Project C: The Clouds — Resources Database");
        sb.AppendLine("# Generated by Tools → ProjectC → Resources → CSV Import/Export");
        sb.AppendLine("# This file is the source of truth. .asset files are GENERATED from it.");
        sb.AppendLine();
        
        // 1. inventory
        sb.AppendLine("# block=inventory");
        sb.AppendLine(string.Join(",", ResourcesCsvSchema.BlockInventory.Columns.Select(c => c.name)));
        var allItems = Resources.FindObjectsOfTypeAll<ItemData>();  // or AssetDatabase.FindAssets
        foreach (var item in allItems.OrderBy(i => i.itemName))
        {
            sb.AppendLine(string.Join(",", new[] {
                EscapeCsv(item.itemName),
                item.itemType.ToString(),
                EscapeCsv(item.description ?? ""),
                item.maxStack.ToString(CultureInfo.InvariantCulture),
                item.weightKg.ToString("0.0##", CultureInfo.InvariantCulture),
            }));
        }
        sb.AppendLine();
        
        // 2. tradeItems
        sb.AppendLine("# block=tradeItems");
        sb.AppendLine(string.Join(",", ResourcesCsvSchema.BlockTradeItems.Columns.Select(c => c.name)));
        var allTrade = Resources.FindObjectsOfTypeAll<TradeItemDefinition>();
        foreach (var t in allTrade.OrderBy(t => t.itemId))
        {
            sb.AppendLine(...);
        }
        sb.AppendLine();
        
        // 3. marketItems
        sb.AppendLine("# block=marketItems");
        sb.AppendLine(...);
        var allMarketConfigs = Resources.FindObjectsOfTypeAll<MarketConfig>();
        foreach (var mc in allMarketConfigs)
        foreach (var item in mc.items)
        {
            sb.AppendLine(...);
        }
        sb.AppendLine();
        
        // 4. exchangeRates
        sb.AppendLine("# block=exchangeRates");
        sb.AppendLine(...);
        var exchangeConfig = Resources.Load<ExchangeRateConfig>("Exchange/DefaultExchangeRate");
        foreach (var rate in exchangeConfig.rates)
        {
            sb.AppendLine(...);
        }
        sb.AppendLine();
        
        // 5. recipes — Phase 2 (out of scope MVP)
        // (пустая секция — recipes экспортируются отдельным CraftingCsvExporter)
        
        var dir = Path.GetDirectoryName(csvPath);
        if (!string.IsNullOrEmpty(dir) && !Directory.Exists(dir))
            Directory.CreateDirectory(dir);
        File.WriteAllText(csvPath, sb.ToString(), new UTF8Encoding(true));  // BOM
        AssetDatabase.Refresh();
        
        return sb.ToString().Count(c => c == '\n');
    }
}
```

> (со своей схемой и путями). Сейчас не пишется.

## 6. Пример работы (user flow)

### Сценарий 1: Добавить новый ресурс (медь-камень)

1. Content writer открывает `Resources_Import.csv` в Excel.
2. Добавляет строку в `# block=inventory`:
   ```
   Медный камень,Resources,Кусок медной породы. Добывается в шахтах.,30,3.0
   ```
3. Если хочет продавать на рынке — добавляет строки в `# block=tradeItems`,
   `# block=marketItems`, `# block=exchangeRates`.
4. Сохраняет CSV (Ctrl+S).
5. Открывает Unity → `Tools → ProjectC → Resources → CSV Import/Export`.
6. Browse → выбирает CSV → Preview → видит 1 новую строку, 0 errors.
7. Click **Import** → dialog: "Created 1 ItemData, Updated 0. Total 1 changes. 0 errors."
8. Проверяет: `Resources/Items/Item_Resources_Медный_камень.asset` создан,
   `ItemRegistry.asset` содержит новую запись.
9. **Запускает Play Mode** → подбирает PickupItem с этим `ItemData` →
   InventoryWorld загружает → работает.

> **Рецепты (Phase 2):** если нужен крафт — отдельный CSV импортируется через
> `CraftingCsvImporter` (см. `Crafting_system/` roadmap). В этом MVP не покрывается.

### Сценарий 2: Round-trip (export → import)

1. Пользователь делает `Tools → ProjectC → Resources → Export` → сохраняет `dump.csv`.
2. Открывает `dump.csv` в Excel, видит все 47+5+8+4+5 = 69 строк.
3. Правит `weightKg` у 3 предметов.
4. Сохраняет → импортирует обратно.
5. Preview показывает 3 строки в inventory, `weightKg` обновлены.
6. Import → 3 updated, 0 errors.

### Сценарий 3: Боевой (что-то пошло не так)

4. Click Import → эта строка skipped, остальные обрабатываются. Dialog: "Updated 67, skipped 2, 0 errors."

## 7. Подводные камни (что особенно проверить при имплементации)

1. **GUID ассета при CreateAsset:** при первом создании Unity генерирует новый GUID и
   пишет в `.meta`. Если есть ссылки на этот ассет в других `.asset` (LootTable, Recipe),
   они **останутся целыми** — Unity резолвит по GUID при загрузке. **НО** если мы
   пересоздаём ассет (importer удаляет и создаёт заново) — GUID сменится. **Решено:**
   importer **обновляет** существующие (если есть), создаёт только если нет.

2. **Один ItemData — много ItemType:** валидация enum в `Enum.TryParse<ItemType>(row.Get("itemType"))` —
   если CSV содержит "Resource" (без 's'), парсер не пройдёт. CrossValidate выдаст ошибку
   с сообщением "Unknown itemType 'Resource'". OK.

3. **Duplicate `itemName` в inventory (case-insensitive):** если две строки
   "Железная руда" и "железная руда" — это один и тот же ресурс. **Решено:** `HashSet<string>(StringComparer.OrdinalIgnoreCase)`.

4. **MarketConfig `items` list capacity:** при `items.Add(new MarketItemConfig {...})` —
   если уже есть такой (itemId, locationId), **обновляем** поля, иначе **добавляем**. 
   Использовать цикл с поиском.

5. **ExchangeRateConfig `rates` list:** то же — обновляем или добавляем.

6. **ItemRegistry `entries` list:** `id` уникален, ищем по `id` (если уже зарегистрирован) или
   по `itemName` (если новый item, но почему-то есть Entry — warning).

7. **Reimport уже существующего ассета:** `AssetDatabase.LoadAssetAtPath<T>(path)` — если
   null, создаём. **Проверить:** что `path` нормализован (forward slashes, не backslashes).

8. **SanitizeFileName для русских имён:** `Item_Resources_Железная_руда.asset` — это OK,
   но проверить, что Unity принимает кириллицу в путях. (На macOS может быть проблема;
   на Windows OK). **Митигация:** использовать `Path.GetInvalidFileNameChars()` для
   замены.

9. **CSV с BOM (Excel) и без (ручной редактор):** парсер `File.ReadAllText` без указания
   encoding может дать BOM в первом поле. **Решено:** `new UTF8Encoding(true)` + strip
   leading `\uFEFF` в `SplitCsvLines`.

10. **Параллельные репозитории для кеша:** `Resources.FindObjectsOfTypeAll<ItemData>()` —
    находит ВСЕ ItemData, включая те, что НЕ в Resources/. **Решено:** фильтровать по
    `AssetDatabase.GetAssetPath(item).StartsWith("Assets/_Project/Resources/Items/")`.

11. **TradeItemDefinition vs TradeDatabase:** все `TradeItemDefinition` лежат в
    `Trade/Data/Items/`, а `TradeDatabase.allItems` хранит **ссылки** на них. **При создании
    нового TradeItemDefinition надо добавить его в `TradeDatabase.allItems`.**

12. **MarketConfig.locationId = "primium"** — это конвенция, но я НЕ нашёл существующих
    `MarketConfig_primium.asset` в проекте. **Сделано:** при импорте создаём файл
    `MarketConfig_{locationId}.asset` если нет. **Уточнить у пользователя:** имена
    locationId (primium/secundus/tertius/quartus — из `MarketConfig.cs:18`).

## 11. Альтернативы, которые НЕ рекомендуются

### 11.1 Несколько отдельных CSV (по подсистеме)

`inventory.csv`, `tradeItems.csv`, ... — каждый импортируется отдельно. **Минусы:**
- 5 разных файлов = 5 разных source of truth, противоречия гарантированы.
- Content writer должен синхронизировать itemName вручную.
- Exchanger и Recipe — кросс-системные, их нельзя «отделить».

### 11.2 JSON / YAML вместо CSV

**Минусы:** не открывается в Excel, content writer'у нужны технические знания.
Прецедент проекта — CSV (QuestCsv). **Единообразие важнее**.

### 11.3 Прямое редактирование `.asset` (YAML) в git

**Минусы:**
- GUID-ы конфликтуют при merge (UUID-collision в разных ветках = потеря ассета).
- Невозможно «посмотреть что есть» без парсинга YAML.
- Нет validation (опечатка в `itemType` = silent fail).
- **Запрещено** AGENTS.md ("Never write `.meta` files by hand").

### 11.4 ScriptableObject addressables / AddressableAssetSettings

Фаза 3+, не для Stage 2.5. Сейчас все SO — прямые ссылки.
