# Bug Report — Сессия A3: Проблемы масштаба и позиционирования гор

## Summary
**Title**: Горы выглядят мелкими и кучными из-за смешения систем координат (метры vs scaled units)
**ID**: BUG-A3-001
**Severity**: S1-Critical (блокер для всего ландшафта)
**Priority**: P1-Immediate
**Status**: Fixed (ожидает верификации в Unity)
**Reported**: 13 апреля 2026
**Reporter**: A3 Session

## Classification
- **Category**: Visual / Game World
- **System**: Mountain Generation (PeakDataFiller, MountainMeshBuilder, MountainMassifBuilder)
- **Frequency**: Always (100%)
- **Regression**: Yes — работало неправильно с момента создания системы (сессии A1, A2)

## Environment
- **Build**: Unity 6, ветка `qwen-gamestudio-agent-dev`
- **Scene**: Mountain Generation Scene
- **Game State**: После запуска "Build All Mountain Meshes"

## Reproduction Steps
**Preconditions**: 
1. Запустить PeakDataFiller → Fill All Massifs (29 пиков)
2. Запустить MountainMassifBuilder → Build All Mountain Meshes

**Ожидаемый результат**: 
- 29 гор разбросаны по карте (расстояния 3000-8700 units между главными пиками)
- Каждая гора видна как гора (не точка, не блинчик)
- Эверест — самая высокая и массивная
- Формы различимы (4 типа)
- Материалы правильные по биому

**Фактический результат (ДО исправления)**:
- Горы все мелкие, странных форм
- Расположены кучно, близко к "старту"
- Не на правильных координатах
- Визуально невозможно различить типы гор

## Technical Context

### Root Cause: Смешение двух систем координат

**Документация (WorldLandscape_Design.md) говорит:**
- X,Z = scaled units (1:2000 от реальности)
- Y = scaled units (meters / 100)
- Пример: Эверест (0, 88.48, 0)

**PeakDataFiller.cs (ДО исправления) заполнял:**
- X,Z = scaled units ✅ (правильно)
- Y = метры БЕЗ деления на 100 ❌ (НЕПРАВИЛЬНО!)
- Пример: Эверест (0, 8848, 0) — Y в 100x больше!

**Результат:**
```
Эверест worldPosition = (0, 8848, 0)  // Y = 8848 вместо 88.48!
meshHeight = 8848 * 0.01 * 5 = 442.4  // Меш маленький относительно Y
gameObjectPosition.y = 8848 - 221.2 = 8626.8  // Гора в небе на высоте 8627!
```

**Почему горы "кучно":**
- X,Z координаты правильные (3000-8700 units расстояния)
- Но Y всех гор = 3000-8800 (вместо 30-88)
- Визуально с камеры на Y=10-50 горы выглядят как точки вдалеке в "небе"
- Камера не может нормально приблизиться из-за far clip plane

### Исправленные файлы

| Файл | Что исправлено |
|------|---------------|
| `PeakDataFiller.cs` | Y координаты всех 29 пиков разделены на 100 (8848 → 88.48), snowLineY тоже (5500 → 55.00) |
| `MountainMeshBuilder.cs` | Убран heightScale (0.01), формула: `meshHeight = worldPosition.y * meshHeightMultiplier` |
| `MountainMassifBuilder.cs` | Синхронизировано с MountainMeshBuilder, убрана зависимость от heightScale |

### Новая формула (после исправления)

```
// Эверест:
worldPosition = (0, 88.48, 0)
meshHeight = 88.48 * 5.0 = 442.4 units
gameObjectPosition = (0, 88.48 - 221.2, 0) = (0, -132.7, 0)
Вершина меша (world) = -132.7 + 442.4 = 88.48 ✅
Основание меша (world) = -132.7

Расстояние Эверест → Монблан:
ΔX = 1310, ΔZ = 2810, distance = 3100 units ✅
```

## Notes

### Почему это НЕ было замечено раньше

1. Сессии A1, A2 фокусировались на компиляции и создании мешей
2. Материалы работали — визуально цвета были правильные
3. Компиляция без ошибок создала ложное ощущение работоспособности
4. Пользователь тестирует в Unity и сообщил: "горы все мелкие странных форм, расположены кучно"

### Системная проблема

Документация и код использовали разные системы координат:
- **Документация**: Y = meters / 100 (WorldLandscape_Design.md §2.1)
- **Код**: Y = meters (PeakDataFiller.cs)

Это привело к 100x расхождению в Y координатах.

### Рекомендации на будущее

1. **Всегда проверять единую систему координат** при создании новых систем
2. **Добавить validation** в PeakDataFiller: если Y > 1000, warning о возможном неправильном масштабе
3. **Документировать единицы измерения** в XML doc каждого поля Vector3
