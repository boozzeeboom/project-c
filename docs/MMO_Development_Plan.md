## Этап 2: Сетевой фундамент (Недели 7-10)
**Цель:** Реализовать клиент-серверное взаимодействие.

### Задачи:
1. **Серверная часть:**
   - Архитектура: Авторитарный сервер (клиент только отправляет ввод)
   - Реализация:
     - WebSocket сервер на .NET 8 (или Mirror Dedicated Server)
     - Система комнат/лобби для групп до 4 игроков
     - Синхронизация позиций, анимаций, состояний объектов

2. **Клиентская интеграация:**
   - Замена локального контроллера на сетевой (NetworkBehaviour)
   - Предсказание движения (Client-side Prediction)
   - Интерполяция других игроков (Lerp + буферизация)

3. **Тестирование сети:**
   - Локальный запуск: 1 сервер + 2-3 клиента
   - Проверка синхронизации физики кораблей
   - Обработка разрывов соединения (переподключение)

**Результат:** Работающий мультиплеер с 2-4 игроками в одной сцене.

## Критические риски и меры:
1. **Сложность сетевой синхронизации физики**
   → Начать с упрощенной модели, тестировать с первых недель.

2. **Читеры в экономике**
   → Вся критичная логика только на сервере, логирование действий.

3. **Высокие требования к инфраструктуре**
   → Использовать облачные решения (AWS GameLift, Azure PlayFab) для старта.

4. **Перегрузка контентом**
   → Фокус на минимально жизнеспособный продукт (MVP) для раннего доступа.
