## 1. 🔴 Критические баги (5 шт.)

### B1. CollectRpc — отсутствует owner-guard (Anti-grief)

| Поле | Значение |
|------|----------|
| **Где** | `CraftingServer.cs:327-374` |
| **Риск** | **Любой игрок может украсть чужой готовый крафт.** В мультиплеере — дыра. |
| **Тикеты** | Нет тикета (не добавлен) |
| **Суть** | CollectRpc проверяет только `job.State == Completed`, но НЕ проверяет `job.OwnerClientId`. |
| **Воспроизведение** | Игрок A завершил крафт. Игрок B подходит к станции и забирает результат. |
| **Фикс** | Добавить проверки: `if (job.OwnerClientId != clientId || job.State != CraftingJobState.Completed) → Denied(NotOwner)`. |

**Код (текущий, проблема):**
```csharp
// CraftingServer.cs ~327
[Rpc(SendTo.Server)]
public void CollectRpc(ulong stationNetId, RpcParams rpcParams = default)
{
    // ... validate station exists, get job
    // ❌ НЕТ: if (job.OwnerClientId != clientId) { Denied(NotOwner); return; }
    if (job.State != CraftingJobState.Completed)
    {
        Denied(stationNetId, clientId, CraftingResultCode.NotStarted, "Ещё не готово");
        return;
    }
    // ... process collect
}
```

#### Шаг 1.3: B3 — валидация buffer→recipe

**Файл:** `CraftingServer.cs` (новый метод) + `RecipeData.cs` (утвердить API)

```csharp
private bool ValidateBufferMatchesRecipe(CraftingJob job, RecipeData recipe, out string reason)
{
    if (job.Buffer.Count != recipe.Ingredients.Count)
    {
        reason = $"Требуется {recipe.Ingredients.Count} ингредиентов, в буфере {job.Buffer.Count}";
        return false;
    }
    // Сравниваем: сортируем по itemId, проверяем quantity
    var bufferLookup = new Dictionary<int, int>();
    foreach (var b in job.Buffer)
    {
        bufferLookup.TryGetValue(b.itemId, out var cur);
        bufferLookup[b.itemId] = cur + b.quantity;
    }
    foreach (var ing in recipe.Ingredients)
    {
        int itemId = CraftingWorld.GetItemId(ing.item); // или через InventoryWorld
        if (!bufferLookup.TryGetValue(itemId, out var qty) || qty < ing.quantity)
        {
            reason = $"Не хватает {ing.item?.itemName ?? "ингредиента"} (нужно {ing.quantity}, в буфере {qty})";
            return false;
        }
    }
    reason = null;
    return true;
}
```

Вызвать в `StartCraftRpc` перед мутацией.

**Edge-cases:**
- buffer пустой → "Буфер пуст"
- buffer больше чем нужно → "Лишние ингредиенты"
- buffer меньше → "Не хватает ..."
- buffer не соответствует рецепту (другие предметы) → "Неверные ингредиенты"

**Время:** 45 мин

### 11.4 Что протестировать в Play Mode

**Критический путь (обязательно):**

1. **B1 — Защита от кражи:** Host крафтит → ждёт завершения. Client (второй игрок) подходит к станции → нажимает Collect → **должен получить отказ** «Только заказчик может забрать результат». Host нажимает Collect → получает предметы.
2. **B2 — Отмена без дублирования:** Host кладёт ингредиенты, запускает крафт → отменяет. Проверить инвентарь: ресурсы вернулись **ровно один раз** (не задвоились).
3. **B3 — Валидация буфера:** Положить в буфер «не те» ингредиенты (не соответствующие рецепту) → нажать Start → **должен быть отказ** с сообщением о несовпадении.
4. **B4 — Инструмент:** Настроить станцию с MetaRequirement (требуется инструмент X). Без инструмента в инвентаре → Start → **отказ**. С инструментом → Start → **успех**.
5. **B5 — SpeedMultiplier:** Поставить `CraftSpeedMultiplier = 2.0` на конфиге станции → крафт должен идти в 2 раза быстрее.
6. **T4 — Отмена чужим:** Host запустил крафт → Client пытается нажать Cancel → **отказ** «Только заказчик может отменить».
7. **T5 — Сбор при ошибке:** Симулировать успешный Collect (предметы выданы, станция сброшена в Empty).

**Регресс (желательно):**

8. **Полный флоу:** Открыть станцию → выбрать рецепт → добавить ингредиенты (+1 / Все) → Start → наблюдать прогресс-бар → Completed → Collect → ресурсы в инвентаре.
9. **Client-side UI:** На чистом клиенте открыть окно крафта — не должно быть ошибок `CraftingWorld.IsInitialized == false`.
10. **Консоль:** 0 error-логов с упоминанием `[CraftingServer]`, `[CraftingStation]`, `[CraftingWorld]`.
