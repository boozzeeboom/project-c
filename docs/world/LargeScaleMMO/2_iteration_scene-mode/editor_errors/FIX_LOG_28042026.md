## ИСПРАВЛЕНИЯ: BootstrapSceneGenerator.cs

### 1. EmptyScene вместо DefaultGameObjects (P0) ✅

**Файл:** `BootstrapSceneGenerator.cs:78`

**Проблема:** `NewSceneSetup.DefaultGameObjects` создаёт дефолтную камеру с AudioListener, затем `CreateCameraSystem()` создаёт ЕЩЁ ОДНУ камеру. Итог: 2 AudioListener в BootstrapScene.


### 2. InputSystemUIInputModule вместо StandaloneInputModule (P0) ✅

**Файл:** `BootstrapSceneGenerator.cs:100-111`

**Проблема:** `StandaloneInputModule` использует старый `UnityEngine.Input` API, который несовместим с Input System Package (activeInputHandler=1 в Player Settings). Это вызывало спам `InvalidOperationException`.

**Исправление:**
```csharp
// БЫЛО:
GameObject eventSystem = new GameObject("EventSystem");
eventSystem.AddComponent<UnityEngine.EventSystems.EventSystem>();
eventSystem.AddComponent<UnityEngine.EventSystems.StandaloneInputModule>();

// СТАЛО:
var existingEventSystem = UnityEngine.EventSystems.EventSystem.FindObjectOfType<UnityEngine.EventSystems.EventSystem>();
if (existingEventSystem != null) return; // duplicate protection

GameObject eventSystem = new GameObject("EventSystem");
eventSystem.AddComponent<UnityEngine.EventSystems.EventSystem>();
var inputModule = eventSystem.AddComponent<UnityEngine.EventSystems.InputSystemUIInputModule>();
```

### 4. NetworkTestMenu вместо lambda OnClick (P0) ✅

**Файл:** `BootstrapSceneGenerator.cs:300-358`

```csharp
// БЫЛО:
```

## Root Cause Analysis: Почему ошибки каскадировались

```
Генератор сцен (BootstrapSceneGenerator)
    │
    ├─ CreateEventSystem() ─────────────────┐
    │                                       │
    │   (DefaultGameObjects создаёт        │
    │    дефолтную камеру + EventSystem)    │
    │                                       │
    ├─ CreateNetworkTestCanvas() ───────────┤
    │                                       │
    │   (создаёт ВТОРОЙ EventSystem!)      │
    │                                       │
    ├─ CreateCameraSystem() ───────────────┤
    │                                       │
    │   (создаёт ВТОРУЮ камеру с            │
    │    AudioListener!)                   │
    │                                       │
    └─ CreateButton() с lambda ─────────────┘
                                            │
                 (lambda не сериализуется,
                  кнопки не работают)
```

- `EmptyScene` - никаких дефолтных объектов
- Один EventSystem с правильным input module
- Одна камера с AudioListener

## ДОПОЛНИТЕЛЬНЫЕ ИСПРАВЛЕНИЯ: Компиляция (29.04.2026)

### 8. InputSystemUIInputModule через reflection (P0) ✅

**Ошибка:** `The type or namespace name 'InputSystemUIInputModule' does not exist in the namespace 'UnityEngine.EventSystems'`

**Причина:** `InputSystemUIInputModule` находится в сборке `Unity.InputSystem.UI`, а не в `UnityEngine.EventSystems`.

**Исправление:** Использовать reflection для добавления компонента:

```csharp
var inputModuleType = GetTypeByName("UnityEngine.InputSystem.UI.InputSystemUIInputModule");
if (inputModuleType != null)
{
    eventSystem.AddComponent(inputModuleType);
}
else
{
    eventSystem.AddComponent<UnityEngine.EventSystems.StandaloneInputModule>();
}

private static Type GetTypeByName(string typeName)
{
    foreach (var assembly in System.AppDomain.CurrentDomain.GetAssemblies())
    {
        var type = assembly.GetType(typeName, false);
        if (type != null) return type;
    }
    return null;
}
```
