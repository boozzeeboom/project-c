## Day 2 (05.05.2026) - CLOUDENGINE Analysis

### Problem: Flat Appearance

User feedback: VeilShader.shader works but looks flat — "куски плоского тумана плывут". Wanted "вихри, закрученные, пустоты и полости" like CLOUDENGINE.

### CLOUDENGINE Deep Analysis

Analyzed `cloud_advanced.frag` from `C:\CLOUDPROJECT\CLOUDENGINE`:

**Why CLOUDENGINE looks volumetric:**
1. Volumetric accumulation over 100+ ray steps
2. Density gradient normals — lighting reacts to 3D density
3. Rim lighting from view direction
4. Ghibli cel-shading

**Key insight:** "вихри и пустоты" come from lighting, not complex noise. Simple 2D FBM + proper lighting = volumetric look.

### Solution: VeilRaymarchMesh.shader

Created new shader with CLOUDENGINE-style lighting:
- `CalcNormal()` — density gradient normal calculation
- `CelShade()` — Ghibli-style quantized diffuse
- Rim lighting with view-dependent power
- Volumetric accumulation with Beer-Lambert

## ⚠️ KNOWN LIMITATION

**Diving Under Veil NOT Supported:**

Current implementation uses horizontal plane at Y=800-1200. When player goes BELOW veil (Y < 800):
- Player is inside volumetric layer
- Plane is above player
- No inner surface to display
- Result: empty space

**User requirement:** "若玩家在云层下方，就看不到内部的云壁"

**Solution needed (future):** Cylinder approach with inner surface rendering.
