# Roadmap Page Redesign — 2026-07-31

**Задача:** Переработать `docs/roadmap.html` для новой аудитории.
Проблема: страница разрослась — новичок не понимает, с чего начать.
Решение: 2 главные сворачиваемые секции **Code** + **Visual** с крупным прогрессом.

## Структура

1. **Header** (как есть) — общий прогресс + коммит
2. **💻 Code Development** — прогресс-бар + сворачиваемый список всех code-задач
3. **🎨 Visual/Art** — прогресс-бар + сворачиваемый список всех visual-задач
4. **📋 Полный roadmap** — кнопка, открывающая все детальные phase cards (прежняя структура)
5. **Остальные секции** (GDD, Graph, Collab, CloudEngine) — только в полном режиме

## Классификация задач

Каждая задача из MMO_Development_Plan.md определяется как code или visual
по эвристике (ключевые слова в тексте задачи + заголовок секции).

**Visual-ключевые:** шейдер, модель, анимация, персонаж, облака, материал, текстура,
пост-процессинг, bloom, skybox, moon, constellation, кастомизация, Customisation,
Equipment Visual, арт, Ghibli, меш, mesh, иконка, префаб (визуальный).

**Code — всё остальное.**

## Технические изменения

- Удалены group-tabs (phases/extra/cloudengine/gdd/all)
- Phase cards скрыты за кнопкой «Полный roadmap»
- JS: новая функция `classifyTask()` + агрегация по двум категориям
- CSS: минимальные добавки (main-card стили)
- Остальное (коллаб, граф, GDD) — сохранено, но скрыто по умолчанию

## Verification

Открыть `docs/roadmap.html` в браузере:
- Видны 2 большие карточки Code (~80-90%) и Visual (~10-20%)
- Клик разворачивает список задач
- Кнопка «Полный roadmap» показывает все детальные этапы
- Общий прогресс сверху корректен
