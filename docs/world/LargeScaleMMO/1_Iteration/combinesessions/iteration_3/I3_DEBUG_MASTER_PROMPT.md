### 🚨 NEW ISSUE: FIX I3-002 - Chunk Loading Around Spawn (Not Player)

**Проблема:** Чанки грузятся вокруг спавн зоны (0,0,0) вместо реальной позиции игрока.

**Симптомы:**
1. Игрок отходит на 100,000 от спавна
2. Нажимает F7 - чанки загружаются у спавна, не у игрока
3. HUD показывает "loaded chunks 20" у спавна
4. Игрок отходит ещё дальше - чанки выгружаются, loaded chunks = 0
5. F7 - чанки опять грузятся у спавна

**Причина:** `WorldStreamingManager.UpdateStreaming()` использовал `Camera.main` вместо позиции игрока.
- Камера под TradeZones (сдвигается с FloatingOriginMP)
- Позиция камеры = (0,0,0) после сдвига мира

**Решение:** Добавлены новые методы в `WorldStreamingManager`:
1. `GetPlayerPositionForChunking()` - получает позицию игрока для стриминга
2. `FindLocalPlayerTransform()` - надёжный поиск локального игрока через NGO

**Изменения в коде:**
- `UpdateStreaming()` теперь использует `GetPlayerPositionForChunking()` вместо `Camera.main`
- `UpdatePreload()` теперь использует `GetPlayerPositionForChunking()` вместо `Camera.main`
- Добавлено кэширование позиции игрока (поиск каждую секунду, не каждый кадр)

**Документация:** `I3_CHUNK_LOADING_FIX.md`

## История: Iteration 3

### FIX I3-001 (Offset Growth) - RESOLVED

**Проблема:** Глобальный offset продолжает расти, даже когда персонаж стоит на месте.
**Причина:** `GetWorldPosition()` возвращает неправильную позицию без учёта `_totalOffset`.

### Симптомы из логов
```
[FloatingOriginMP] GetWorldPosition: using NetworkPlayer IsOwner=(-98410, 3, 480578)
[FloatingOriginMP] SERVER SHIFT: offset=(-100000.00, 0.00, 480000.00), cameraPos=(-98410.39, 3.00, 480578.20)
[FloatingOriginMP] SERVER SHIFT complete: totalOffset=(-3060000.00, 0.00, 16430000.00)
```

### Анализ проблемы

1. **Игрок стоит на месте** → позиция не меняется: `(-98410.39, 3.00, 480578.20)`
2. **WorldRoot уже сдвинут** → позиция: `(3060000.00, 0.00, -16430000.00)`
3. **Реальная мировая позиция игрока** = игрок + WorldRoot offset:
   - X: -98410 + 3060000 = 2,961,590
   - Z: 480578 + (-16430000) = 16,432,578
4. **Но FloatingOriginMP думает** что игрок на (-98410, 3, 480578)
5. **Сдвиг применяется** на (-100000, 0, 480000) → offset растёт бесконечно!
