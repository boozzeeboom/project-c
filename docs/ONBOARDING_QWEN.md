## Краткая сводка проекта

**Project C: The Clouds** — MMO-игра на Unity 6 по мотивам книги «Интеграл Пьявица».
Сеттинг: 2090 год, человечество живёт на горных вершинах над облаками, корабли на антигравии курсируют между городами.
Визуальный стиль: **Sci-Fi + Ghibli** — промышленный дизайн + мягкие облака, градиенты, объёмный свет.

**Текущее состояние:** Полностью рабочий сетевой мультиплеер (Host+Client), пешее управление, корабли с кооп-пилотированием, инвентарь с сундуками, процедурная генерация мира.

## Архитектура проекта

### Сетевая модель

```
HOST (Unity Editor / билд)
├── СЕРВЕР (авторитарный) — единственный источник истины
└── Клиент 0 (игрок-хост)

Клиент 1, 2, N (билды или Unity)
  ↓ UnityTransport (UDP, порт 7777)
```

**Ключевой принцип:** Сервер — единственный источник истины. Клиенты шлют ввод → сервер считает → реплицирует результат.

### Основные скрипты

| Скрипт | Назначение | Ключевые методы |
|--------|-----------|-----------------|
| `NetworkManagerController.cs` | Обёртка NGO, обработка подключений | `StartHost()`, `StartClient()`, `OnClientConnectedCallback` |
| `NetworkPlayer.cs` | Игрок: движение, камера, инвентарь | `ProcessMovement()`, `OnNetworkSpawn()`, `SpawnCamera()` |
| `ShipController.cs` | Корабль: физика, кооп-пилотирование | `SubmitShipInputRpc()`, `FixedUpdate()`, `AddPilotRpc()` |
| `NetworkUI.cs` | UI сети: Disconnect кнопка, панель подключения | `CreateDisconnectButton()`, `UpdateConnectionUI()` |
| `ThirdPersonCamera.cs` | Орбитальная камера от третьего лица | `SetTarget()`, `LateUpdate()` |
| `Inventory.cs` | Инвентарь: хранение по типам, группировка | `AddItem()`, `AddMultipleItems()` |
| `InventoryUI.cs` | Круговое колесо (8 секторов, GL-рендер) | `RenderWheel()`, `TriggerSectorFlash()` |
| `ChestContainer.cs` | Сундук с LootTable, анимация | `Open()`, `TriggerLoot()` |
| `WorldGenerator.cs` | Процедурная генерация: 15 пиков, 890+ облаков | `Generate()`, `GeneratePeaks()` |

### Каталог `Assets/_Project/`

```
Assets/_Project/
├── Art/              # 3D-модели, текстуры
├── InputActions/     # Input System ассеты
├── Items/            # ScriptableObject предметов
├── Material/         # Материалы
├── Prefabs/          # Префабы (NetworkPlayer, камеры, корабли)
├── Scenes/           # Сцены (ProjectC_1.unity — основная)
└── Scripts/
    ├── Core/         # Ядро: сеть, генерация, инвентарь
    ├── Player/       # Контроллеры: пешеход, корабль
    └── UI/           # Интерфейс: NetworkUI, InventoryUI, подсказки
```

## Текущие стандарты и соглашения

### 🔴 КРИТИЧНО: НЕ ТРОГАТЬ `.meta` ФАЙЛЫ

Unity автоматически создаёт `.meta` файлы для каждого ассета. Ручное создание/редактирование ломает ссылки.

### Протокол сотрудничества (из QWEN-MASTER-PROMPT.md)

1. **Задавай вопросы** для прояснения контекста
2. **Предлагай 2-4 варианта** с плюсами/минусами
3. **ЖДИ решения** пользователя — НЕ решай за него
4. **Пиши черновик** секции → показывай → жди одобрения
5. **Спрашивай** «Могу ли я записать это в [filepath]?» перед записью
6. **Записывай** только после явного «да»

### Git Workflow

- Ветка: `qwen-dev` (основная), `upstream = https://github.com/boozzeeboom/project-c.git`
- Коммитить после каждого маленького шага
- Принцип: "Медленнее = Быстрее" — 1 файл/класс за раз

## Типичные ошибки (как не сломать)

| Ошибка | Последствие | Как избежать |
|--------|-------------|--------------|
| Редактирование `.meta` файлов | Сломанные ссылки, ошибки импорта | **НИКОГДА** не трогать `.meta` |
| Большие коммиты | Сложно откатить, ломает работу | 1 файл/класс за раз |
| Изменение NetworkTransform без тестирования | Рассинхронизация по сети | Тестировать Host+Client после каждого изменения |
| Добавление префабов без регистрации в DefaultNetworkPrefabs | Префаб не спавнится у клиентов | Всегда проверять `DefaultNetworkPrefabs.asset` |
| Изменение RPC без обновления всех клиентов | Ошибки сериализации | Согласовывать параметры на всех клиентах |
